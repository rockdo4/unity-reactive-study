# 실습 02 — 클리커 확장 (2부 마무리)

따라하기(교안 06)로 완성한 자신의 클리커에 기능을 추가함. **난이도 순**이므로 위에서부터. 제출물: 스크립트 + 실행 영상 + **Tracker 누수 0 스크린샷**(과제 D).

## 과제 A. 피버 쿨다운 남은 시간 표시 ★

**요구사항** — 피버 발동 후, 피버 버튼 라벨에 남은 쿨다운을 초 단위로 표시 (`피버 (12초)` → … → `피버!`). 쿨다운이 끝나면 원래 문구로.

힌트:

- 피버 발동 스트림에서 `SelectMany`(또는 `Select`+`Switch`)로 `Observable.Interval(1초)`를 시작하고 15회만(`Take(15)`) 받기
- `Scan`이나 `Select((값, 인덱스) => ...)`로 남은 초 계산 — `Index` 연산자도 카탈로그에 있음
- 더 깔끔한 접근: `Observable.Timer(0초, 1초).Take(16).Select(t => 15 - t)` 를 발동 시점마다 새로 구독

채점 포인트: 표시 갱신에 `Update()`/코루틴 미사용 · 쿨다운 종료 시 문구 복원

## 과제 B. 크리티컬 업그레이드 ★★

**요구사항** — 새 업그레이드 "행운"(비용 50, 살 때마다 크리티컬 확률 +10%, 최대 50%). 클릭이 크리티컬이면 골드 2배 획득 + 로그 출력.

힌트:

- Model에 `ReactiveProperty<int> CritChance`와 `TryBuyLuckUpgrade()` 추가 — 2단계(업그레이드) 패턴 복사
- 크리티컬 판정은 `Click()` 안에서 `UnityEngine.Random` 대신 `System.Random`(Model이 UnityEngine을 모르게 유지) — 판정 결과를 반환값이나 별도 `Subject<long>`(크리티컬 발생 스트림)으로 Presenter에 알리기
- UI: 구매 버튼 + 활성화(CombineLatest) + 확률 표시 바인딩 — 전부 배운 패턴

채점 포인트: Model/Presenter 책임 분리 유지 · `TryBuy` 패턴 · 크리티컬 알림이 폴링 없이 스트림으로 전달

## 과제 C. 황금 보너스 ★★★

**요구사항** — 20초마다 "황금 버튼"이 화면에 나타나 3초간 유지. 그 안에 클릭하면 `현재 초당 수입 × 30` 골드 획득, 클릭하거나 3초가 지나면 사라짐.

힌트:

- 등장 주기: `Observable.Interval(20초)` → 버튼 `SetActive(true)`
- 소멸: 등장 후 `Observable.Timer(3초)`와 "클릭" 중 **먼저 온 쪽**이 닫음 — `Merge` + `Take(1)` 또는 `TakeUntil`
- 보상 계산은 Model에 메서드로 (`AddBonus` 재사용 가능)
- 모든 구독의 수명 주의 — 등장할 때마다 새 구독을 만들면 정리는 누가?

채점 포인트: 먼저 온 쪽이 닫는 합성(`Merge`/`TakeUntil`) · 반복 등장에도 구독이 누적되지 않음(Tracker로 확인)

## 과제 D. 누수 0 증명 (필수)

**요구사항** — 위 과제를 모두 붙인 상태에서:

1. `ObservableTracker.EnableTracking = true`로 Play.
2. 충분히 플레이(피버·황금 버튼 포함) 후 자신의 Presenter 오브젝트를 삭제.
3. `Window > Observable Tracker`에서 클리커 관련 구독이 **0개**임을 스크린샷으로 제출.

남아 있다면 StackTrace로 생성 지점을 찾아 고칠 것 — 그 과정 자체가 이 실습의 목적임.

## 보너스 (선택) ★★★★

- **저장/로드** — Model의 값들을 `PlayerPrefs`에 저장하고 시작 시 복원. 로드가 "`.Value` 대입"만으로 UI까지 갱신되는 것을 확인.
- **DPS 측정기** — 최근 5초간 획득 골드를 실시간 표시. 힌트: 획득 이벤트를 `Chunk` 또는 `Scan`+`Timer` 조합으로.
