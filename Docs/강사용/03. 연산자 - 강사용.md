## 1. 강사 사전 학습 · 리마인드

- **이 차시가 강의의 하이라이트** — "리액티브의 진짜 가치 = 연산자 합성"을 몸으로 느끼게 하는 시간. 1차시 명령형 더블클릭(코루틴+플래그+타이머)이 두 줄이 되는 장면을 클라이맥스로 설계할 것.
- **전부 가르치려 하지 말 것** — 연산자는 수십 개고 카탈로그는 사전임. 수업에서 제대로 다루는 건 8개뿐: `Where` `Select` `DistinctUntilChanged` `Scan` / `Debounce` `ThrottleFirst` `Chunk` / `Merge` `CombineLatest`. 나머지는 "그룹에서 찾아 쓰는 법"만.
- **Debounce vs ThrottleFirst를 자기 입으로** — 수업 전 소리 내어 연습: "Debounce는 **끝나면**(잠잠해진 뒤 마지막), ThrottleFirst는 **먼저**(첫 발 통과 후 구간 무시)." 연타 입력 하나를 놓고 두 연산자가 어디서 발화하는지 마블로 그릴 수 있어야 함.
- **Share를 뒤로 미루지 말 것** — 더블클릭 코드에 `Share()`가 들어 있어 학생이 반드시 물음. "같은 스트림을 두 곳(본체+경계)이 쓰니 구독을 하나로 공유"라는 실용 설명으로 충분. 콜드/핫 이론 강의로 빠지지 말 것.
- **UniRx 매핑표 숙지** — 학생이 검색해 오는 코드가 UniRx일 확률이 높음. `Throttle→Debounce`(최대 함정), `Buffer→Chunk`, `.Count→.Length` 세 개는 즉답할 수 있어야 함.

## 2. 수업 진행

### 도입 (~3분)

> 💡 1차시의 명령형 더블클릭 코드를 다시 띄우고 — "오늘 이게 두 줄이 됩니다. 그리고 왜 두 줄이 '짧아서'가 아니라 '의도가 보여서' 좋은지 이야기합니다."

### 1~2장. 파이프라인과 카탈로그 (~7분)

- 체인 = 파이프라인, **구독은 맨 끝 한 번**. 칠판에 `source → Where → Select → Subscribe` 관로 그림.
- 카탈로그 표는 **분류만 훑기** — "필요할 때 그룹에서 찾는다"는 태도 심기.

### 3장. 필터·변환 (~7분)

- `Where`/`Select`는 LINQ 그대로라 빠르게. `DistinctUntilChanged`는 "바뀔 때만 한 번"(UI·사운드) 용례로.
- `Scan`은 마블(`──5─8─18─▶`)로. "2부 클리커의 콤보가 이걸로 만들어진다" 예고.

### 4장. 시간 연산자 (~10분, 핵심)

- Debounce(검색창)·ThrottleFirst(쿨다운) 마블을 칠판에 크게. **같은 입력 스트림**에 두 연산자를 적용한 출력 차이를 학생에게 그리게 하기.
- Chunk(개수·시간·경계) → 더블클릭 파이프라인 완성. "한 문장으로 읽어 보세요"(답: "클릭이 잠잠해질 때까지 모아서, 2개 이상이면").
- **칠판**: `Debounce = 끝나면 · ThrottleFirst = 먼저 · Chunk = 모아서`.

### 5장. 결합 (~5분)

- `Merge`(여러 입력원 → 한 행동), `CombineLatest`(여러 값 → 한 조건). CombineLatest는 "버튼 활성화의 정석"으로 5차시·2부에서 반복됨을 예고.
- `Zip`/`WithLatestFrom`은 이름만 — 카탈로그行.

### 6~7장. 따라하기 — 데모 패널 2 (~15분)

1. 더블클릭: 화면을 천천히 두 번(반응 없음) vs 빠르게 두 번(반응) — Debounce 경계의 의미가 체감됨.
2. 검색창: 타이핑 중에는 침묵 → 멈추면 0.5초 뒤 발화. 같은 검색어 재입력은 무시(DistinctUntilChanged).
3. 3연타·쿨다운: 같은 버튼의 두 파이프라인이 독립적으로 동작함을 강조 — "스트림 하나에 정책 여러 개".
4. `OperatorsDemo.cs`를 같이 읽고, **각 파이프라인의 마블을 손그림**으로 그리게 함(이 차시 최고의 연습).

- **⚠️ 함정**: `Share()` 질문이 나오면 — "clickStream을 본체와 Debounce 경계 두 곳이 써서, 구독을 하나로 공유하는 것"까지만. 깊은 핫/콜드 강의는 7장(교안) 수준으로 제한.
- **⚠️ 함정**: 학생이 UniRx 예제의 `Throttle`을 들고 옴 — 고의 실패 시연 기회. 그대로 입력해 **컴파일 에러**를 보여 주고 매핑표로 해결. 백 마디 설명보다 효과적.

### 정리 (~3분)

- 시간 3총사·결합 2총사 복기. "한 스트림을 두 곳 이상에서 쓰면?" → "Share를 의심."

## 3. 예상 Q&A

> ❓ **연산자를 다 외워야 하나요?**
> 아님. 분류(생성/필터/변환/시간/결합/누적)만 잡고 필요할 때 공식 문서에서 찾는 것. 실무에서 자주 쓰는 건 오늘 배운 8개 안팎.

> ❓ **Debounce랑 ThrottleLast는 뭐가 다른가요?**
> ThrottleLast(=UniRx Sample)는 **고정 주기마다** 구간의 마지막 값을 내보냄(잠잠함과 무관). Debounce는 **잠잠해져야** 내보냄. 입력이 계속되는 동안 ThrottleLast는 주기적으로 나오고 Debounce는 침묵.

> ❓ **체인이 길어지면 디버깅은 어떻게 하나요?**
> ①중간 스트림을 지역 변수로 쪼개 이름을 붙인다 ②`Do`(R3의 사이드 이펙트 연산자)나 중간 `Subscribe`+로그로 단계별 값을 찍는다 ③Tracker로 구독 자체를 확인(4차시). "7~8개 이상 이어지면 설계를 의심"이 교안의 권고.

> ❓ **`InvokeRepeating`으로 주기 반복하던 건 R3로 어떻게 바뀌나요?** (전환형)
> `Observable.Interval(TimeSpan.FromSeconds(N))`이 정확한 대응 — 차이는 반환이 스트림이라 뒤에 `Where`·`Debounce`·`ThrottleFirst`를 바로 이어 "주기 반복 + 시간 가공"을 한 체인으로 표현할 수 있다는 것. 구독은 `AddTo`로 수명 결합. ('기존 방식 → R3' 한눈 매핑표는 [운영 가이드 1장](00.%20강의%20운영%20가이드.md))

## 4. 이해도 퀴즈

1. 같은 연타 입력에 대해 Debounce와 ThrottleFirst가 각각 언제 발화하는지 마블로 그리시오.
2. 더블클릭 파이프라인 `clicks.Chunk(clicks.Debounce(250ms)).Where(xs => xs.Length >= 2)`를 한국어 한 문장으로 읽으시오.
3. "HP > 0이고 탄약 > 0일 때만 발사 버튼 활성화"를 어떤 연산자로 만드는지 코드 골격을 쓰시오.
4. UniRx 코드의 `Buffer`·`Throttle`을 R3로 옮길 때 각각 무엇으로 바꾸는가?

> ✅ **해설**
>
> 1. 입력 `──a─b─c────────`에 대해 Debounce는 잠잠해진 뒤 `c` 한 번, ThrottleFirst는 시작 즉시 `a` 한 번.
> 2. "클릭이 250ms 잠잠해질 때까지 모아서, 2개 이상이면 더블클릭."
> 3. `hp.CombineLatest(ammo, (h, a) => h > 0 && a > 0).DistinctUntilChanged().Subscribe(can => button.interactable = can)`
> 4. `Buffer→Chunk`(결과 `.Count`→`.Length`), `Throttle→Debounce`(R3에 Throttle 없음).
