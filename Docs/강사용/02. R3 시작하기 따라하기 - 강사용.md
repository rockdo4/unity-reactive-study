## 생성법 1: Range — 즉시 흐르고 완료 (6분)

> 🔧 오늘 쓸 부품: 02차시 `Observable.Range`(콜드 스트림).

**함께 작성** — `Start()`의 빈 본문에 첫 구독:

```csharp
// 1) Range — 콜드(cold) Observable: 구독하는 순간 값이 흐르고 즉시 완료된다
m_RangeButton
    .OnClickAsObservable()
    .Subscribe(_ =>
    {
        Log("--- Range(1, 4) 구독 ---");
        Observable
            .Range(1, 4)
            .Subscribe(x => Log($"Range OnNext: {x}"), result => Log($"Range OnCompleted: {result}"))
            .AddTo(this);
    })
    .AddTo(this);
```

- 버튼 클릭이 `OnClickAsObservable()`로 스트림이 됨 — 클릭 스트림 구독 안에서 `Range` 스트림을 만들어 구독(스트림 안의 스트림).
- `Subscribe` 두 번째 인자가 `OnCompleted` 콜백 → 완료 로그. 완료가 있음이 이번 핵심.
- `.AddTo(this)`는 4차시 주제. 지금은 오브젝트 파괴 시 구독 자동 해제로만 이해.

**실행·확인**

1. Play → `Range(1, 4)` 클릭 → 로그에 `Range OnNext: 1`~`4` 연달아, 그리고 `Range OnCompleted: ...`.
2. 버튼 다시 누르면 매번 처음부터 1~4 새로 흐름.

> 📌 핵심: 구독 즉시 4개 다 흐르고 완료(`|`)까지. 누를 때마다 자기 전용으로 새로 흐르는 게 **콜드**. Subscribe는 등록이지만 `Range`는 등록과 동시에 값이 오는 특수 경우(이론 2-2·2-6).

**변형 실습**: `Range(1, 4)`→`Range(1, 10)`으로 핫 리로드 → 1~10 흐름. 직접 한 글자 고쳐 보기.

**이론 연결**: 2-2, 2-3, 2-6.

## 생성법 2: Timer — 2초 뒤 한 번 (4분)

> 🔧 오늘 쓸 부품: 02차시 `Observable.Timer`(일회성 스트림).

**함께 작성** — 이어서 `Start()`에 두 번째 구독:

```csharp
// 2) Timer — 지정 시간 뒤 한 번 발행하고 완료되는 일회성 스트림
m_TimerButton
    .OnClickAsObservable()
    .Subscribe(_ =>
    {
        Log("Timer(2초) 시작...");
        Observable.Timer(TimeSpan.FromSeconds(2)).Subscribe(_ => Log("Timer 발행! (2초 경과)")).AddTo(this);
    })
    .AddTo(this);
```

- `Observable.Timer(TimeSpan.FromSeconds(2))` — 2초 뒤 값 하나 내보내고 완료. 발행 값 안 쓰므로 `_`로 버림.

**실행·확인**

1. Play → `Timer 2초` 클릭 → 즉시 `Timer(2초) 시작...`, 2초 뒤 `Timer 발행! (2초 경과)`.

> 📌 핵심: 2초 동안 화면 안 멈춤(논블로킹). 코루틴 `WaitForSeconds`의 "N초 뒤 1회 실행"이 한 줄로. 일회성이라 한 번 발행하고 끝.

**변형 실습**: `FromSeconds(2)`→`FromSeconds(0.5)`로 지연 체감.

**이론 연결**: 2-3.

## 생성법 3: Interval — 무한, 멈춤은 Dispose (6분)

> 🔧 오늘 쓸 부품: 02차시 `Observable.Interval`(무한 스트림), 구독 보관용 `IDisposable` 필드.

**함께 작성** — 토글 구조가 핵심. 천천히 한 줄씩:

```csharp
// 3) Interval — 일정 주기로 무한 발행. 멈추려면 구독을 직접 Dispose해야 한다
m_IntervalToggleButton
    .OnClickAsObservable()
    .Subscribe(_ =>
    {
        if (m_IntervalSubscription == null)
        {
            int tick = 0;
            m_IntervalSubscription = Observable
                .Interval(TimeSpan.FromSeconds(1))
                .Subscribe(_ => Log($"Interval tick {++tick}"))
                .AddTo(this); // 토글을 깜빡해도 오브젝트 파괴 시 함께 해제되도록 이중 안전망
            Log("Interval 시작 (1초 간격)");
            SetToggleLabel("Interval 중지");
        }
        else
        {
            m_IntervalSubscription.Dispose(); // Dispose가 곧 구독 해제
            m_IntervalSubscription = null;
            Log("Interval 중지 (Dispose 호출)");
            SetToggleLabel("Interval 시작");
        }
    })
    .AddTo(this);
```

- `m_IntervalSubscription`(`IDisposable` 필드)에 구독 보관 — 구독을 변수로 들고 있어야 나중에 멈춤. 이론 2-2 Subscribe 반환값이 구독 그 자체라는 점의 실물. (이 필드와 `SetToggleLabel`은 스켈레톤 제공.)
- `null`이면 시작(구독 만들어 보관), 아니면 `Dispose()`로 멈추고 `null`로 — 중지=`Dispose()`를 코드로 확인.
- `.AddTo(this)`는 토글 깜빡 잊고 Play 정지해도 오브젝트 파괴 시 함께 해제되는 이중 안전망(4차시 예고).

**실행·확인**

1. Play → `Interval 시작` 클릭 → 1초 간격 `Interval tick 1`, `tick 2`, … 계속 쌓임. 라벨이 `Interval 중지`로.
2. 다시 클릭(`Interval 중지`) → `Interval 중지 (Dispose 호출)`. tick 멈춤.

> 📌 핵심: `Interval`은 스스로 끝나지 않는 **무한** 스트림(완료 없음). 멈추는 유일한 방법이 구독 `Dispose()` — 4차시 수명 관리 복선.

**변형 실습**: `FromSeconds(1)`→`FromSeconds(0.3)`으로 tick 빨라지는 것 확인.

> ⚠️ 함정: Interval 여러 번 시작하려는 학생 나옴(토글이라 안 생기지만 코드 고치다 보면). 스트림은 만들 때마다 새것이라는 콜드 개념 설명 기회로.

**이론 연결**: 2-3, 2-6 (그리고 4차시 복선).

## 생성법 4: Subject — 손으로 미는 발행기 (4분)

> 🔧 오늘 쓸 부품: 02차시 `Subject`(직접 미는 발행기).

> 📌 필드는 이미 있음: `Subject` 발행기 필드와 정리(`OnDestroy`)는 스켈레톤 제공(아래는 읽기만). `Start()`에서 구독·발행 두 줄만 작성.

```csharp
// 필드 — 코드 어디서든 OnNext로 값을 직접 밀어 넣는 발행기
private readonly Subject<string> m_Subject = new();
...
private void OnDestroy()
{
    m_Subject.Dispose(); // 발행기(Subject)도 IDisposable
}
```

**함께 작성** — 구독은 `Start`에서 한 번(수신 측), 발행은 버튼에서(송신 측):

```csharp
// 4) Subject — OnNext를 호출해 직접 발행
m_Subject.Subscribe(msg => Log($"Subject 수신: {msg}")).AddTo(this);
m_SubjectButton
    .OnClickAsObservable()
    .Subscribe(_ => m_Subject.OnNext($"메시지 #{++m_SubjectCount}"))
    .AddTo(this);
```

- `m_Subject.Subscribe(...)`로 미리 구독(수신 측), 버튼 핸들러에서 `m_Subject.OnNext(...)`로 발행(송신 측). 하나의 `Subject`가 받기(`OnNext`)도 내주기(`Subscribe`)도 겸함(이론 2-4).
- `Subject`는 발행기 자체가 `IDisposable`이라 `OnDestroy`에서 `Dispose()`(제공됨). 데모는 핫이라 구독 전 누른 값은 못 받음(클릭 스트림과 같은 성질).

**실행·확인**

1. Play → `Subject.OnNext` 클릭 → 누를 때마다 `Subject 수신: 메시지 #1`, `#2`, … 카운트 증가.

> 📌 핵심: 팩토리(시간·횟수가 값을 만듦)와 달리 내가 버튼 누르는 그 순간 발행됨. `OnNext` 호출 시점이 곧 발행 시점.

**변형 실습**: 메시지 문자열을 `$"점수 +{++m_SubjectCount * 10}"` 등으로 바꿔 발행 데이터 가공.

**이론 연결**: 2-4, 2-6.

## 생성법 5: FromEvent — 기존 event를 스트림으로 (4분)

> 🔧 오늘 쓸 부품: 02차시 `Observable.FromEvent`(기존 event 감싸기).

> 📌 event 필드는 이미 있음: 감쌀 기존 C# `event LegacyScoreEvent`는 스켈레톤 선언됨(아래는 읽기만). `Start()`에서 감싸 구독하는 줄과 발행하는 줄만 작성.

```csharp
// 기존 C# event. FromEvent로 Observable로 감싸는 시연용
private event Action<int> LegacyScoreEvent;
```

**함께 작성** — `Start()`의 마지막 구독:

```csharp
// 5) FromEvent — 기존 C# event를 Observable로 감싼다 (구독 해제 시 -=도 자동)
Observable
    .FromEvent<int>(h => LegacyScoreEvent += h, h => LegacyScoreEvent -= h)
    .Subscribe(score => Log($"FromEvent 수신: 점수 {score}"))
    .AddTo(this);
m_EventButton
    .OnClickAsObservable()
    .Subscribe(_ => LegacyScoreEvent?.Invoke(m_LegacyScore += 10))
    .AddTo(this);
```

- `FromEvent<int>(h => e += h, h => e -= h)` — 구독 시작 시 `+=`, 구독 해제(`Dispose`) 시 `-=` 자동. 시그니처(`h => e += h`)는 외워 쓰는 관용구(이론 2-5).
- 버튼은 평범한 C# `event` 발생(`LegacyScoreEvent?.Invoke(...)`) — 기존 코드 그대로인데 그걸 스트림으로 받는 게 핵심.

**실행·확인**

1. Play → `event 발생` 클릭 → `FromEvent 수신: 점수 10`, `20`, … 10씩 증가.

> 📌 핵심: 화면 동작은 Subject와 비슷하지만 안에서는 기존 C# `event`를 스트림으로 감싼 것. 기존 이벤트 코드를 점진적으로 리액티브에 합류시키는 다리.

**변형 실습**: 증가폭 `+= 10`→`+= 25`로 점수 흐름 변화 확인.

**이론 연결**: 2-5.

# 🎓 마무리 (5분)

## 만들기 5종 요약

| 생성법         | 핵심                    | 마블           |
| -------------- | ----------------------- | -------------- |
| `Range(1, 4)`  | 즉시 흐르고 완료(콜드)  | `-1-2-3-4-\|`  |
| `Timer(2s)`    | N초 뒤 한 번(일회성)    | `------o-\|`   |
| `Interval(1s)` | 무한 — 멈춤은 `Dispose` | `--o--o--o-->` |
| `Subject`      | `OnNext` 시점마다 발행  | `--o----o-o->` |
| `FromEvent`    | 기존 event 감싸기       | event 발생 시  |

- 핵심 둘: ① 완료(`|`)가 있는 스트림과 없는 스트림, ② 무한 스트림(`Interval`)은 `Dispose()`로만 멈춤.
- 구독은 등록일 뿐, 값을 **언제** 흘리는지가 생성법마다 다름.

## 마무리 Q&A 예상 질문

- **Subject vs ReactiveProperty?** 현재값이 의미 있으면 RP(체력·골드), 순간의 사건이면 Subject(공격 명중). 데모는 Subject만 다룸, RP 실물(바인딩)은 5차시.
- **`AddTo(this)`가 계속 붙던데?** 구독을 오브젝트 수명에 묶는 것 — 4차시 주제. 지금은 파괴 시 자동 해제로만 이해.
- **`Interval`을 안 멈추면?** 핸들 없이 만든 무한 구독은 누수 — 4차시에서 Tracker로 눈으로 확인.

- 씬이 꼬이면 강사 프로젝트는 `Tools > Reactive Study > Create All Scenes`로 재생성, 학생 프로젝트는 패키지 재임포트(작성 중 코드는 백업 안내).
- 코드는 복붙 말고 타이핑 유도 — 직접 친 줄만 남음. 에러 나면 그 자리에서 함께 디버깅(에러도 교보재). 놓친 학생에겐 모범 코드(`CreateStreamsDemo.cs`) 해당 부분 화면 공유하되 따라 치게 할 것.
- 생성법별 시간 늘어지면 변형 실습 건너뛰고 함께 작성 + 실행·확인만. 질문은 각 생성법 종료 후로 모음.
