- [ ] **이 차시가 강의 핵심** — 연산자 합성으로 의도를 그대로 코드에 적는 체감. 1차시 명령형 더블클릭(코루틴+플래그+타이머)이 두 줄로 줄어드는 장면을 핵심 장면으로.
- [ ] **전부 가르치지 말 것** — 카탈로그는 사전. 제대로 다루는 건 8개: `Where` `Select` `DistinctUntilChanged` `Scan` / `Debounce` `ThrottleFirst` `Chunk` / `Merge` `CombineLatest`. 나머지는 "그룹에서 찾아 쓰는 법"만.
- [ ] **Debounce vs ThrottleFirst 입으로 연습** — Debounce는 **끝나면**(잠잠해진 뒤 마지막), ThrottleFirst는 **먼저**(첫 발 통과 후 구간 무시). 연타 입력 하나로 두 연산자 발화 지점 마블 그릴 수 있어야 함.
- [ ] **UniRx 매핑 숙지** — 학생이 들고 오는 코드는 UniRx일 확률 높음. `Throttle→Debounce`(최대 함정), `Buffer→Chunk`, `.Count→.Length` 즉답.

## 3-1. 파이프라인과 카탈로그 (7분)

- 연산자(operator) = **Observable을 받아 새 Observable을 돌려주는 메서드**. 그래서 메서드 체인으로 파이프라인.
- 체인 = 파이프라인, **구독은 맨 끝 한 번**. 중간 단계는 전부 설계도, `Subscribe`가 호출돼야 물이 흐름.

**판서** — `source → Where → Select → Subscribe` 관로 그림:

```csharp
source                      // 원본 스트림
    .Where(x => x > 0)      // 거르고
    .Select(x => x * 10)    // 바꾸고
    .Subscribe(Print);      // 마지막에 구독
```

```
source: --(-1)--2--0--3-->
Where:  --------2-----3-->     (0보다 큰 것만)
Select: --------20----30-->    (×10)
```

**카탈로그 — 분류만 훑기**: "필요할 때 그룹에서 찾는다"는 태도. 외우는 게 아니라 분류만.

| 그룹          | 대표 연산자 (R3 기준)                                             | 하는 일                      |
| ------------- | ----------------------------------------------------------------- | ---------------------------- |
| **생성**      | `Range` · `Timer` · `Interval` · `FromEvent` · `EveryUpdate`      | 스트림을 만듦 (2차시)        |
| **필터**      | `Where` · `Distinct` · `DistinctUntilChanged` · `Take` · `Skip`   | 조건·개수로 걸러냄           |
| **변환**      | `Select` · `Cast` · `Index`                                       | 값을 다른 모양으로           |
| **시간**      | `Debounce` · `ThrottleFirst` · `ThrottleLast` · `Delay` · `Chunk` | 시간으로 다듬음              |
| **결합**      | `Merge` · `CombineLatest` · `Zip` · `WithLatestFrom` · `Concat`   | 여러 스트림을 합침           |
| **누적**      | `Scan` · `Aggregate`                                              | 흘러온 값을 누적함           |
| **에러/완료** | `OnErrorResume` 처리 · `Catch` · `OnErrorResumeAsFailure`         | 에러·완료를 다룸 (5차시)     |
| **공유**      | `Share` · `Replay` 계열                                           | 하나의 구독을 여럿이 나눠 씀 |

## 3-2. 필터와 변환 — Where · Select · DistinctUntilChanged · Scan (7분)

- LINQ와 같은 이름, 같은 의미. 대상이 "시간 위의 스트림"일 뿐. `Where`/`Select`는 LINQ 그대로라 빠르게 넘김.

**보여줄 코드**:

```csharp
// 10점 단위 점수만 통과시켜 마일스톤 알림
playerStat.OnScoreChanged
    .Where(score => score % 10 == 0)
    .Subscribe(score => Debug.Log($"마일스톤! {score}"));

// HP를 "구간 라벨"로 변환하고, 구간이 바뀔 때만 갱신
vitals.Hp
    .Select(hp => hp < 30 ? "위험" : hp < 70 ? "주의" : "안전")
    .DistinctUntilChanged()                 // 같은 라벨이 연속되면 무시
    .Subscribe(state => m_StateText.text = state);
```

- `DistinctUntilChanged` = **연속 중복 제거**. "바뀔 때만 한 번"(UI 갱신·사운드) 반응의 단골.

**Scan** — "지금까지의 누적"을 흘려보냄(LINQ `Aggregate`의 실시간 버전). 마블로 보여 줄 것:

```csharp
// 획득 골드 스트림 → 누적 합계 스트림
goldGained.Scan(0L,                          // seed(시작값) 0
        (total, gained) => total + gained)   // (직전 누적, 새 값) → 새 누적. 매번 흘려보냄
    .Subscribe(total => m_TotalText.text = $"누적 {total}");
```

```
in:   --5---3---10-->
Scan: --5---8---18-->    (seed 0에서 시작해 누적)
```

- 2부 클리커 콤보가 이걸로 만들어짐을 예고.

## 3-3. 시간 연산자 — Debounce · ThrottleFirst · Chunk (10분, 핵심)

- 이 차시의 핵심. **상태 플래그·타이머를 직접 들지 않고** 시간 조건을 선언.
- Debounce(검색창)·ThrottleFirst(쿨다운) 마블 크게 판서 → **같은 입력 스트림**에 두 연산자 적용한 출력 차이를 학생이 직접 그리게.

### Debounce — 잠잠해질 때까지 기다리기

마지막 이벤트 후 지정 시간 동안 새 이벤트가 없으면, 그 마지막 값을 내보냄.

```
in:   --a-b--c----------d----->
Debounce(0.5s):
out:  ------------c----------d-->   (연타 중에는 침묵, 잠잠해지면 마지막 값)
```

```csharp
// 검색창: 타이핑이 0.5초 멈췄을 때만, 직전과 다른 검색어일 때만 검색 실행
m_SearchInput.onValueChanged.AsObservable()
    .Debounce(TimeSpan.FromMilliseconds(500))   // 0.5초 잠잠해지면 마지막 값 한 번(=입력 끝)
    .DistinctUntilChanged()                     // 직전 검색어와 같으면 무시
    .Subscribe(query => Search(query));
```

- 쓰임: 검색 자동완성, 설정 자동 저장, 리사이즈 후 재배치 — "**입력이 끝났을 때** 한 번".

### ThrottleFirst — 첫 발만 통과, 구간 무시

첫 이벤트를 통과시키고, 지정 시간 동안 나머지를 무시.

```
in:   --a-b-c----d-e-------f-->
ThrottleFirst(1s):
out:  --a--------d---------f-->   (발동 직후 1초는 귀를 닫음)
```

```csharp
// 스킬 버튼: 첫 클릭만 발동, 2초 쿨다운
m_SkillButton.OnClickAsObservable()
    .ThrottleFirst(TimeSpan.FromSeconds(2))
    .Subscribe(_ => CastSkill());
```

- 쓰임: 스킬 쿨다운, 연타 방지, 공격 입력 제한 — "**먼저 발동하고** 잠시 무시".
- **Debounce는 "끝나면", ThrottleFirst는 "먼저"**. 같은 연타 입력에 Debounce는 마지막 한 번, ThrottleFirst는 처음 한 번 반응.

### Chunk — 모아서 묶기

이벤트를 모았다가 **경계**(시간·개수·다른 스트림)가 오면 배열로 내보냄.

```csharp
// 500ms 안에 3번 클릭 → 3연타
taps.Chunk(TimeSpan.FromMilliseconds(500), 3)
    .Where(xs => xs.Length >= 3)
    .Subscribe(_ => Debug.Log("3연타!"));

// 더블클릭: "클릭이 250ms 잠잠해질 때"를 경계로 클릭을 묶고, 2개 이상이면
var clicks = Observable.EveryUpdate()
    .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame)
    .Share();                                            // ← 3-5에서 설명

clicks
    .Chunk(clicks.Debounce(TimeSpan.FromMilliseconds(250)))   // 경계 = 잠잠해진 순간
    .Where(xs => xs.Length >= 2)
    .Subscribe(_ => Debug.Log("더블 클릭"));
```

- 더블클릭 파이프라인 완성 후 "한 문장으로 읽기" 유도(답: "클릭이 잠잠해질 때까지 모아서, 2개 이상이면"). 1차시 명령형 더블클릭(코루틴+플래그+타이머)과 비교 — **의도가 그대로 코드**.

**칠판 정리**: `Debounce = 끝나면 · ThrottleFirst = 먼저 · Chunk = 모아서`.

> 🔁 **Unity에서 늘 하던 방식 ↔ R3 (시간·입력 다루기)** — `Update`와 코루틴으로 손수 짜던 입력·시간 로직이 시간 연산자로 무엇이 되는지의 지도.
>
> | Unity에서 늘 하던 방식                          | 무엇이 문제였나                           | R3 대응                                                      |
> | ----------------------------------------------- | ----------------------------------------- | ------------------------------------------------------------ |
> | `Update`에서 매 프레임 입력 폴링 + 조건 분기    | 안 바뀐 프레임도 매번 검사, 조건이 흩어짐 | `EveryUpdate()` + `Where` — 조건을 스트림 한 곳에 선언       |
> | 코루틴+플래그+타이머로 더블클릭·연타 판정       | 상태 플래그·타이머 누적 → 경계·리셋 버그  | `Chunk` + `Debounce`(경계) — "모아서 N개 이상"이 그대로 코드 |
> | `if (Time.time - last > cd)`로 쿨다운·연타 방지 | 시각 비교 변수 수동 관리, 재사용 어려움   | `ThrottleFirst` — "먼저 한 번, 구간 무시"를 한 줄로          |
>
> 메타 축 둘: **폴링(pull) → 스트림 구독(push)**, **명령형 how(플래그·타이머) → 선언형 what(연산자 합성)**.

## 3-4. 결합 연산자 — Merge · CombineLatest (5분)

### Merge — 여러 스트림을 한 줄로

```
A:      --a-----c----->
B:      -----b-----d-->
Merge:  --a--b--c--d-->    (도착 순서대로 한 스트림에)
```

```csharp
// 키보드 스페이스든 버튼 클릭이든 "발사"는 발사
var fireByKey = Observable.EveryUpdate()
    .Where(_ => Keyboard.current != null && Keyboard.current.spaceKey.wasPressedThisFrame)
    .Select(_ => Unit.Default);                 // 두 스트림의 타입을 Unit으로 통일(Merge하려면 같은 타입)
var fireByButton = m_FireButton.OnClickAsObservable();

fireByKey.Merge(fireByButton)                   // 둘 중 어느 쪽이 와도 한 스트림으로
    .ThrottleFirst(TimeSpan.FromSeconds(0.5))   // 합친 뒤 공통 쿨다운 적용
    .Subscribe(_ => Fire());
```

- Merge = "여러 입력원 → 한 행동".

### CombineLatest — 각 스트림의 "최신값" 조합

어느 쪽이든 새 값이 오면, **모든 스트림의 최신값**을 모아 내보냄.

```
HP:              --100--------70------------------30------->
MP:              --50-------------------10----------------->
CombineLatest:   --(100,50)---(70,50)---(70,10)---(30,10)-->
```

```csharp
// HP가 있고 MP가 충분할 때만 스킬 버튼 활성화 — "조건이 여러 값에 걸칠 때"의 정석
hp.CombineLatest(mp, (h, m) => h > 0 && m >= 20)  // HP·MP 어느 쪽이 바뀌든 둘의 최신값으로 조건 평가
    .DistinctUntilChanged()                       // 결과(true/false)가 바뀔 때만
    .Subscribe(canUse => m_SkillButton.interactable = canUse);
```

- CombineLatest = "버튼 활성화의 정석". **5차시·2부에서 반복**됨을 예고. 대부분의 게임 UI는 `CombineLatest`로 충분.
- `Zip`(짝이 맞을 때만)·`WithLatestFrom`(한쪽이 주도)은 이름만 — 카탈로그 행.

## 3-5. Share와 UniRx 이름 매핑 (8분)

### Share — 핫/콜드 혼동 방지

- 더블클릭 코드에서 `clicks`를 **본체와 Debounce 경계 두 곳**이 씀. `Share()` 없으면 두 곳이 **각자 따로 구독**해 `EveryUpdate` 검사가 두 벌 돎(콜드 동작). 값비싼 스트림(네트워크·무거운 계산)이면 버그.

```csharp
var clicks = rawClicks.Share();   // 원본을 한 번만 구독하고, 구독자들이 그 결과를 나눠 받음
```

- 규칙: 한 스트림을 두 곳 이상에서 구독하면 콜드 재실행(중복 구독) 의심 → `Share()`로 묶을지 검토.

> ⚠️ **함정**: `Share()` 질문 나오면 "clicks를 본체와 Debounce 경계 두 곳이 써서 구독을 하나로 공유"까지만. 깊은 핫/콜드 강의로 빠지지 말 것(콜드/핫 이론은 02차시 수준 제한).

### 옛 UniRx 코드를 읽을 때 (이름 매핑) — 강사 배경지식 / 따라하기 고의 실패 시연용

> 📌 이 매핑표는 **교안에서 제외**(노션 교안 03에 없음). 강사 배경지식 + 따라하기 "고의 실패 시연" 근거로만. 학생 화면(교안)에는 띄우지 말고 필요 시 구두로.

- 인터넷 튜토리얼 다수가 UniRx 기준. R3는 연산자 이름을 .NET LINQ 표준에 맞춰 바꿨으므로 **그대로 붙이면 컴파일 안 됨**(따라하기에서 **고의 실패 시연**으로 각인).

| UniRx                           | R3                            | 함정/메모                                                             |
| ------------------------------- | ----------------------------- | --------------------------------------------------------------------- |
| `Buffer`                        | `Chunk`                       | 결과가 `IList<T>`(`.Count`) → **`T[]`(`.Length`)** — 개수 검사도 바뀜 |
| `Throttle`                      | `Debounce`                    | **최대 함정** — R3엔 `Throttle`이 아예 없음                           |
| `Sample`                        | `ThrottleLast`                |                                                                       |
| `StartWith`                     | `Prepend`                     |                                                                       |
| `ThrottleFirst`                 | `ThrottleFirst`               | 이름·동작 그대로                                                      |
| `IObservable<T>`/`IObserver<T>` | `Observable<T>`/`Observer<T>` | 인터페이스 → 추상 클래스                                              |
| `this.UpdateAsObservable()`     | `Observable.EveryUpdate()`    | R3 전역 스트림은 수명 관리 필수(4차시)                                |

- 핵심 프레임: **개념은 그대로, 이름이 LINQ 표준에 맞춰 바뀜**.

| 개념       | 한 줄 요약                                                               |
| ---------- | ------------------------------------------------------------------------ |

## 💡 강사 팁

- 3-3 시간 연산자가 핵심 — 같은 연타 입력에 Debounce/ThrottleFirst 발화 지점을 학생이 직접 마블로 그리게 하면 가장 잘 남음.
- 카탈로그(3-1 표)는 "다 외우려 들지 말 것" 거듭 강조 — 분류 감각만.
- 시간 늘어지면 `Merge`(3-4 앞부분)는 메모로 끝내고 따라하기로 넘겨도 무방. 단 `CombineLatest`는 5차시·2부 복선이므로 반드시 짚을 것.

## 부록 — 예상 Q&A

> ❓ **연산자를 다 외워야 하나?**
> 아님. 분류(생성/필터/변환/시간/결합/누적)만 잡고 필요할 때 공식 문서에서 찾음. 실무에서 자주 쓰는 건 오늘 배운 8개 안팎.

> ❓ **Debounce랑 ThrottleLast는 뭐가 다른가?**
> ThrottleLast(=UniRx Sample)는 **고정 주기마다** 구간의 마지막 값을 내보냄(잠잠함과 무관). Debounce는 **잠잠해져야** 내보냄. 입력이 계속되는 동안 ThrottleLast는 주기적으로 나오고 Debounce는 침묵.

> ❓ **체인이 길어지면 디버깅은 어떻게 하나?**
> ①중간 스트림을 지역 변수로 쪼개 이름 붙임 ②`Do`(R3 사이드 이펙트 연산자)나 중간 `Subscribe`+로그로 단계별 값 찍음 ③Tracker로 구독 자체 확인(4차시). "7~8개 이상 이어지면 설계 의심"이 교안 권고.

> ❓ **`InvokeRepeating`으로 주기 반복하던 건 R3로 어떻게 바뀌나?** (전환형)
> `Observable.Interval(TimeSpan.FromSeconds(N))`이 정확한 대응 — 차이는 반환이 스트림이라 뒤에 `Where`·`Debounce`·`ThrottleFirst`를 바로 이어 "주기 반복 + 시간 가공"을 한 체인으로 표현 가능. 구독은 `AddTo`로 수명 결합. ('기존 방식 → R3' 한눈 매핑표는 [00. 강사 사전 학습 가이드](00.%20강사%20사전%20학습%20가이드.md))
