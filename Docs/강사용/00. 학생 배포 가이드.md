# 00. 학생 배포 가이드

> 📦 학생에게 수업 자료 배포하는 방법. 이 저장소는 강사 전용, 학생은 클론하지 않음.
> 차시별 진행은 강사용 01~06, 데모 씬 생성·실행은 [씬 생성·패키지 도구 가이드](../데모/씬%20생성·패키지%20도구%20가이드.md) 참조.

---

## 1. 배포 모델 개요

| 자료               | 배포 방법                                                                                                                                                                             |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 교안 (이론)        | **노션 공유 페이지**로 공유 (`Docs/교안/` 01~05의 내용을 노션에 옮겨 게시)                                                                                                            |
| 데모 (씬·스크립트) | **유니티 패키지** — 필수 2종 ① 1부 기초 통합(공통 폰트 포함) ② 2부 클리커, + 선택 1종 ③ 타이밍 바(capstone 스캐폴드). 학생이 빈 URP 프로젝트에 R3·UniTask·Input System 설치 후 임포트 |

- 배포 단위 = 강의 필수 두 패키지(1·2부) + 선택 한 패키지(③ 타이밍 바). 1부(R3 기초)는 5섹션을 한 씬에 담은 **통합 데모 씬 하나**, 2부는 클리커 게임 씬 하나.
- ③ 타이밍 바는 자력 완성 capstone(07)의 **UI 제공 스캐폴드 대안** — 화면·와이어링은 주고 R3만 채우게 함. 배포할 때만 익스포트(상세 [07](07.%20타이밍%20바%20과제%20-%20강사용.md)).
- 학생 패키지 원본 = `Assets/StudentPackage/`. 배포용 씬 3개(`Scenes/`)와 `My*` 스크립트 9종(`Scripts/`). `Tools > Reactive Study > Student Package` 메뉴로 생성·익스포트(3절).
- 수업 시작 때 1부 패키지 먼저 임포트(공통 폰트 포함). 이후 교안(이론) → 통합 씬 해당 섹션 데모를 차시별로 오가며 진행. 2부 진입 시 클리커 패키지 임포트.
- 1부 통합 씬도 **스켈레톤**(라이브 코딩). 씬·UI·Inspector 참조 연결은 완료, 각 데모의 R3 스트림 생성 메서드 본문만 비어 있음. 5섹션(생성·연산자·수명·바인딩·비동기)을 차시별로 한 줄씩 라이브 코딩, 학생이 빈 메서드 채우고 중간중간 Play 확인.
- 2부(클리커)도 **스켈레톤**. 씬·UI·Inspector 연결과 게임 규칙은 제공, 핵심 R3 메서드 본문만 비어 있음. 라이브 코딩 → 학생 따라 입력 → Play 확인.
- 배포 `My*` 코드는 **주석 전부 제거**. 안내는 코드 주석 아닌 따라하기 문서·강의로.
- 클래스명은 강사용 원본 충돌 피해 **`My` 접두사**(`MyCreateStreamsDemo` 등). 학생에게 `My` = "강사와 함께 완성해 가는 코드(1·2부)" 의미.

## 2. 패키지 구성

| 패키지 파일 (`.unitypackage`)          | 부       | 포함 에셋                                                                                                                                                                                | 학생 코드    |
| -------------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| `R3Lecture-Student-1-Basics`           | 1부      | `Assets/Fonts`(Pretendard 한글 폰트) · `Assets/TextMesh Pro`(TMP 설정) · `ReactiveBasics.unity` + `MyCreateStreamsDemo`·`MyOperatorsDemo`·`MyLifetimeDemo`·`MyBindingDemo`·`MyAsyncDemo` | **스켈레톤** |
| `R3Lecture-Student-2-Clicker`          | 2부      | `ReactiveClicker.unity` + `MyClickerModel.cs` · `MyClickerPresenter.cs`                                                                                                                  | **스켈레톤** |
| `R3Lecture-Student-3-TimingBar` (선택) | capstone | `ReactiveTimingBar.unity` + `MyTimingBarModel.cs` · `MyTimingBarPresenter.cs`                                                                                                            | **스켈레톤** |

- 한글 폰트·TMP 설정은 **1부 기초 패키지에만** 포함. 빠지면 한글 깨짐 — 구성 유지하고 학생에게 **1부 패키지 가장 먼저** 임포트 안내(4절). 2부 클리커·③ 타이밍 바는 1부 폰트 그대로 사용.
- ③ 타이밍 바는 capstone(07)을 **UI 제공 스캐폴드로 낼 때만** 배포(자력 완성으로 내면 미배포). 핵심 차이: 클리커는 빈 메서드(`BindInput`/`BindView`)를 미리 줬지만, 타이밍 바는 **UI·와이어링만** 주고 `Start()`만 비워 메서드 구조까지 학생이 설계.
- `MyLifetimeProbe`는 `MyLifetimeDemo.cs` 안에 함께 포함, 1부 패키지로 같이 배포함.
- 1부 통합 씬(`ReactiveBasics.unity`) = 5섹션(생성·연산자·수명·바인딩·비동기) 2열 배치 한 씬. 스크립트는 섹션별 `My*` 5종.

### 스켈레톤이 비워 둔 곳 (1부·2부)

- 두 배포본 모두 **R3 스트림 생성 메서드 본문만 비운 스켈레톤**(주석 제거).
- 씬 Play하면 에러 없이 뜨지만, 비운 메서드 연결 버튼·표시는 코드 채우기 전까지 무반응이 정상(강사가 라이브 코딩으로 채움).

**1부(`ReactiveBasics.unity`)** — 각 데모 구독 메서드가 비어 있음:

| 파일                     | 비어 있는 곳                                                                               |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| `MyCreateStreamsDemo.cs` | `Start()` — Range·Timer·Interval·Subject·FromEvent 다섯 생성법 구독                        |
| `MyOperatorsDemo.cs`     | `SetupDoubleClick`·`SetupSearchDebounce`·`SetupTapAndCooldown`(더블클릭·검색·3연타·쿨다운) |
| `MyLifetimeDemo.cs`      | `Start()`(관리·누수·프로브 트리거 구독) · `MyLifetimeProbe.Start()`(AddTo 프로브 구독)     |
| `MyBindingDemo.cs`       | `Start()` — 바인딩 3패턴(값→표시·파생 상태·CombineLatest)과 버튼 핸들러                    |
| `MyAsyncDemo.cs`         | `Start()` — `SubscribeAwait` Sequential·Drop                                               |

> 1부: 필드·표시 헬퍼(`Log`/`SetText` 등)·생명주기(`Awake`/`OnDestroy`)는 제공. 학생은 빈 `Start()`/`Setup*` 본문만 채움.

**2부(`ReactiveClicker.unity`)** — 핵심 R3 메서드 본문이 비어 있음:

| 파일                    | 비어 있는 곳                                                                                              |
| ----------------------- | --------------------------------------------------------------------------------------------------------- |
| `MyClickerModel.cs`     | 생성자의 `CanBuy*` `CombineLatest` 파생 + 자동수입 `Interval` 스트림 · `TryStartFever`(`Timer` 자동 종료) |
| `MyClickerPresenter.cs` | `BindInput`(클릭 `Share`·콤보·피버 쿨다운·구매 구독) · `BindView`(ReactiveProperty → UI 바인딩)           |

> 클리커 패키지 임포트 후 Play해도 클릭 시 골드 안 오르는 게 정상. 수업에서 `BindInput`/`BindView`와 Model 생성자 함께 채우면 동작. 모범답안 = `Assets/Scripts/ClickerGame/`.

**③ 타이밍 바(`ReactiveTimingBar.unity`, 선택)** — UI·와이어링만 주고 본문 전체가 비어 있음:

| 파일                      | 제공                                                 | 비어 있는 곳                                                |
| ------------------------- | ---------------------------------------------------- | ----------------------------------------------------------- |
| `MyTimingBarModel.cs`     | `ReactiveProperty` 3개·존/점수 상수·생성자·`Dispose` | `ZoneOf`·`Judge`·`ApplyHit`·`ResetForNewGame` 규칙 본문     |
| `MyTimingBarPresenter.cs` | `[SerializeField]` 11개(UI 연결)·빈 `Start()`        | 마커·존·입력 판정·표시·라운드 흐름 스트림 전부, `OnDestroy` |

> 1·2부 스켈레톤과 달리 **빈 메서드 골격을 안 줌** — `Start()`만 비워 두고 스트림 분리·메서드 이름까지 학생이 설계. 모범답안 = `Assets/Scripts/TimingBar/`. 운영·난이도는 [07](07.%20타이밍%20바%20과제%20-%20강사용.md).

## 3. 강사: 패키지 생성·익스포트 절차

1. **학생용 씬 생성(갱신)**: `Tools > Reactive Study > Student Package > Create All Student Scenes`
    - 강사용 SceneCreator로 통합 기초 씬·클리커 씬을 메모리에서 빌드 후 데모 컴포넌트를 `My*` 타입으로 교체(`SwapComponent`) → `Assets/StudentPackage/Scenes/`의 `ReactiveBasics.unity`·`ReactiveClicker.unity`에 저장(같은 경로 덮어써 갱신).
    - 강사용 씬(`Assets/Scenes/`)은 변경 안 됨. 학생용 씬은 이 프로젝트 Build Settings에 추가 안 됨.
    - 컴포넌트 스왑 = 원본 직렬화 필드를 `SerializedProperty` 경로 기준 복사. **`My*`에 원본과 같은 이름 필드 없으면 명시적 예외로 씬 생성 실패**(조용히 깨지지 않음 — 5절 동기화 규칙).
2. **익스포트**: `Tools > Reactive Study > Student Package > Export All Student Packages` → 저장 폴더 선택 → 패키지 3종(`1-Basics`, `2-Clicker`, `3-TimingBar`) 생성.
    - 익스포트 직전 각 에셋 경로를 GUID로 검증. **학생용 씬을 먼저 생성하지 않았거나 파일 이동·개명 시** "패키지에 넣을 에셋이 없습니다" 예외. 1번 메뉴 먼저 실행했는지 확인.
    - 한 패키지만 재배포 시 같은 메뉴의 `Export 1. Basics` / `Export 2. Clicker` / `Export 3. TimingBar`로 개별 익스포트.
    - ③ 타이밍 바는 **capstone을 스캐폴드로 낼 때만** 배포(자력 완성으로 내면 `1`·`2`만 익스포트해도 됨).
3. **배포 전 점검 체크리스트**:
    - [ ] 강사용 런타임 코드(`Assets/Scripts/`)나 씬 구성 고친 적 있으면 대응 `My*` 스크립트에도 반영했는가 (5절 운영 규칙)
    - [ ] `Create All Student Scenes`로 학생용 씬 재생성했는가
    - [ ] 코드 변경 후 Unity Console에 컴파일 에러 없는가 (Unity MCP `Unity_GetConsoleLogs`)
    - [ ] (권장, 배포 전 1회) 별도 빈 URP 프로젝트에 4절 절차대로 임포트해 두 씬 Play 확인 — 특히 1부 패키지 먼저 넣어 한글 안 깨지는지

> ℹ️ CLI/배치: `StudentPackageCreator.BuildAndExportBatch()` 호출 시 씬 생성 후 `STUDENT_PACKAGE_OUTPUT` 환경변수 폴더로 전체 익스포트(씬 생성 실패 시 비정상 종료).

## 4. 학생: 프로젝트 셋업 절차 (수업 전 안내)

아래 순서를 학생에게 안내(노션 교안에 같은 내용 게시 권장). 설치 절차 전체 설명 = [교안 02 §1](../교안/02.%20R3%20시작하기%20-%20Observable%20만들기.md).

1. **빈 URP 프로젝트 생성**: Unity 6.3 **Universal 3D (URP) 템플릿**으로 새 프로젝트.
    - ⚠️ Built-in 템플릿이면 머티리얼 깨짐(분홍색). 반드시 URP 템플릿 사용.
2. **R3 설치** — 두 부분(R3 본체 = 범용 .NET 라이브러리, NuGet으로 / 유니티 통합 = git URL로).
    1. **NuGetForUnity 설치**: `Window > Package Manager` → `+` → _Install package from git URL..._ →
       `https://github.com/GlitchEnzo/NuGetForUnity.git?path=/src/NuGetForUnity`
    2. **R3 본체 설치**: 메뉴 `NuGet > Manage NuGet Packages`에서 `R3` 검색 → Install (`Microsoft.Bcl.TimeProvider` 등 의존 패키지 자동 설치).
    3. **R3.Unity 설치**: Package Manager → git URL →
       `https://github.com/Cysharp/R3.git?path=src/R3.Unity/Assets/R3.Unity#<버전>`
       (`#<버전>` 태그로 코어 NuGet 버전과 일치시켜 고정. 최신은 [R3 릴리스 페이지](https://github.com/Cysharp/R3/releases) 확인.)
    - 설치 확인: 아무 스크립트에서 `using R3;` 컴파일되면 완료.
3. **UniTask 설치**: Package Manager → `+` → _Install package from git URL..._ →
   `https://github.com/Cysharp/UniTask.git?path=src/UniTask/Assets/Plugins/UniTask#2.5.10`
    - 1부 비동기 패널(`MyAsyncDemo`)이 단발 비동기를 UniTask로 작성. 설치 확인: `using Cysharp.Threading.Tasks;` 컴파일되면 완료. (최신은 [UniTask 릴리스](https://github.com/Cysharp/UniTask/releases) 확인.)
4. **Input System 설치**: `Window > Package Manager` → Unity Registry에서 **Input System** 설치 → "Enable the new input backends?" 팝업 **Yes**(에디터 재시작). 데모 씬 `EventSystem`이 `InputSystemUIInputModule`을 써서 New 입력 핸들러 필요.
5. **패키지 임포트**: 배포받은 `.unitypackage` 더블클릭(또는 `Assets > Import Package > Custom Package...`) → 전체 선택 → Import.
    - **`R3Lecture-Student-1-Basics`(1부)를 수업 시작 때 가장 먼저** 임포트. 한글 폰트·TMP 설정이 이 패키지에 있어 건너뛰면 한글 깨짐.
    - 2부 진입 시 `R3Lecture-Student-2-Clicker` 임포트.
6. **확인**: `Assets/StudentPackage/Scenes/ReactiveBasics` 열고 Play → 한글 정상 표시 + Console 컴파일 에러 없으면 셋업 완료.
    - **1부·2부 씬 모두 스켈레톤**이라 Play해도 버튼 무반응이 정상(2절). 코드는 수업에서 강사와 함께 채움. 셋업 확인 기준 = "씬 열리고 한글 안 깨지는가".

## 5. 운영·동기화 규칙

- **동기화**: 강사용 런타임 코드(`Assets/Scripts/`)나 씬 구성 변경 시 → 대응 `Assets/StudentPackage/Scripts/`의 `My*` 같이 수정 → `Create All Student Scenes`로 씬 재생성 → 해당 패키지 재익스포트.
- **필드명 유지**: `My*` 직렬화 필드 이름·접근제어자는 원본과 동일해야 함. 씬 생성 도구(`StudentPackageCreator`)가 필드명 기준 Inspector 참조 복사, 어긋나면 씬 생성 명시적 예외 실패(조용히 깨지지 않음).
- **GUID 검증**: 익스포트 직전 각 에셋 경로 GUID 확인. 학생용 씬 미재생성 또는 파일 이동·개명 시 익스포트 예외 중단 — 1번(씬 생성) → 2번(익스포트) 순서 준수.
- **씬이 깨졌을 때**: 강사 프로젝트는 `Create All Student Scenes`로 재생성, 학생 프로젝트는 패키지 재임포트로 초기 상태 복귀(작성 코드도 덮어써지니 학생 작성 중 코드는 백업 안내).
- **수업 중 학생이 못 따라올 때(2부)**: 모범답안 = `Assets/Scripts/ClickerGame/`. 화면 공유로 해당 메서드만 보여 주고 따라 입력하게 할 것.
