# 00. 학생 배포 가이드

> 📦 이 문서는 **학생에게 수업 자료를 배포하는 방법**을 다룹니다. 이 저장소는 **강사 전용** 프로젝트이며, 학생은 저장소를 클론하지 않습니다.
> 차시별 수업 진행 방법은 강사용 01~06 문서를, 데모 씬 생성·실행은 [씬 생성·패키지 도구 가이드](../데모/씬%20생성·패키지%20도구%20가이드.md)를 참조하세요.

---

## 1. 배포 모델 개요

| 자료               | 배포 방법                                                                                                                                            |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| 교안 (이론)        | **노션 공유 페이지**로 공유 (`Docs/교안/` 01~05의 내용을 노션에 옮겨 게시)                                                                           |
| 데모 (씬·스크립트) | **유니티 패키지 2종(.unitypackage)** — ① 1부 기초 통합(공통 폰트 포함) ② 2부 클리커. 학생이 빈 URP 프로젝트에 R3·UniTask·Input System 설치 후 임포트 |

- 배포 단위는 **두 패키지**입니다. 1부(R3 기초)는 5개 섹션을 한 씬에 담은 **통합 데모 씬 하나**이고, 2부는 클리커 게임 씬 하나입니다.
- 학생 패키지의 원본은 `Assets/StudentPackage/`입니다 — 배포용 씬 2개(`Scenes/`)와 `My*` 스크립트 7종(`Scripts/`)이 모여 있고, `Tools > Reactive Study > Student Package` 메뉴로 생성·익스포트합니다(3절).
- **수업 시작 때 1부 패키지를 먼저 임포트**하고(공통 폰트가 들어 있음), 이후 교안(이론) → 통합 씬의 해당 섹션 데모를 차시별로 오가며 진행합니다. 2부에 들어갈 때 클리커 패키지를 임포트합니다.
- **1부 통합 씬은 완성본**입니다(실행·관찰 중심). 학생은 씬을 열고 Play해 5개 섹션(생성·연산자·수명·바인딩·비동기)의 스트림이 흐르는 모양을 관찰합니다.
- **2부(클리커)는 스켈레톤**입니다 — 씬·UI·Inspector 참조 연결은 모두 완료된 상태이고, **핵심 R3 메서드 본문만 비어** 있습니다. 강사가 설명하며 라이브 코딩하면 학생이 따라 입력하고 중간중간 Play로 확인합니다.
- 클래스 이름은 강사용 원본과의 충돌을 피해 **`My` 접두사**를 씁니다(`MyCreateStreamsDemo` 등). 학생 입장에서는 "내가 실행하고(1부) / 완성해 가는(2부) 코드"라는 의미입니다.

## 2. 패키지 구성

| 패키지 파일 (`.unitypackage`) | 부  | 포함 에셋                                                                                                                                                                                | 학생 코드    |
| ----------------------------- | --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| `R3Lecture-Student-1-Basics`  | 1부 | `Assets/Fonts`(Pretendard 한글 폰트) · `Assets/TextMesh Pro`(TMP 설정) · `ReactiveBasics.unity` + `MyCreateStreamsDemo`·`MyOperatorsDemo`·`MyLifetimeDemo`·`MyBindingDemo`·`MyAsyncDemo` | **완성본**   |
| `R3Lecture-Student-2-Clicker` | 2부 | `ReactiveClicker.unity` + `MyClickerModel.cs` · `MyClickerPresenter.cs`                                                                                                                  | **스켈레톤** |

- 한글 폰트·TMP 설정은 **1부 기초 패키지에만** 들어갑니다. 빠지면 학생 화면에서 한글이 깨지므로 이 구성을 유지하고, 학생에게는 **1부 패키지를 가장 먼저** 임포트하도록 안내하세요(4절). 2부 클리커 패키지는 1부에서 들어온 폰트를 그대로 씁니다.
- `MyLifetimeProbe`는 `MyLifetimeDemo.cs` 안에 함께 들어 있어 1부 패키지로 같이 배포됩니다.
- 1부 통합 씬(`ReactiveBasics.unity`)은 5개 섹션(생성·연산자·수명·바인딩·비동기)을 2열로 배치한 한 씬입니다. 스크립트는 섹션별 `My*` 5종입니다.

### 2부 스켈레톤이 비워 둔 곳

`ReactiveClicker.unity`(2부)는 핵심 R3 메서드 본문만 비운 스켈레톤입니다. **씬을 Play하면 에러 없이 뜨지만, 비워 둔 메서드에 연결된 버튼·표시는 무반응인 것이 정상**입니다(강사가 라이브 코딩으로 채우는 부분). 비어 있는 곳:

| 파일                    | 비어 있는 곳                                                                                              |
| ----------------------- | --------------------------------------------------------------------------------------------------------- |
| `MyClickerModel.cs`     | 생성자의 `CanBuy*` `CombineLatest` 파생 + 자동수입 `Interval` 스트림 · `TryStartFever`(`Timer` 자동 종료) |
| `MyClickerPresenter.cs` | `BindInput`(클릭 `Share`·콤보·피버 쿨다운·구매 구독) · `BindView`(ReactiveProperty → UI 바인딩)           |

> 학생에게 미리 알려 주세요: **"클리커 패키지를 임포트하고 Play하면 클릭해도 골드가 안 오르는 게 정상"** — 수업에서 `BindInput`/`BindView`와 Model 생성자를 함께 채우면 동작합니다. 모범답안은 강사 프로젝트의 `Assets/Scripts/ClickerGame/`에 있습니다.

## 3. 강사: 패키지 생성·익스포트 절차

1. **학생용 씬 생성(갱신)**: `Tools > Reactive Study > Student Package > Create All Student Scenes`
    - 강사용 SceneCreator로 통합 기초 씬과 클리커 씬을 메모리에서 빌드한 뒤 데모 컴포넌트를 `My*` 타입으로 교체(`SwapComponent`)해 `Assets/StudentPackage/Scenes/`의 `ReactiveBasics.unity`·`ReactiveClicker.unity`에 저장합니다(같은 경로에 덮어써 갱신).
    - 강사용 씬(`Assets/Scenes/`)은 변경되지 않으며, 학생용 씬은 이 프로젝트의 Build Settings에 추가되지 않습니다.
    - 컴포넌트 스왑은 원본의 직렬화 필드를 `SerializedProperty` 경로 기준으로 복사합니다. **`My*`에 원본과 같은 이름의 필드가 없으면 명시적 예외로 씬 생성이 실패합니다**(조용히 깨지지 않음 — 5절 동기화 규칙).
2. **익스포트**: `Tools > Reactive Study > Student Package > Export All Student Packages` → 저장 폴더 선택 → 패키지 2종(`1-Basics`, `2-Clicker`)이 생성됩니다.
    - 익스포트 직전 각 에셋 경로를 GUID로 검증합니다 — **학생용 씬을 먼저 생성하지 않았거나 파일이 이동·개명되면** "패키지에 넣을 에셋이 없습니다" 예외가 납니다. 1번 메뉴를 먼저 실행했는지 확인하세요.
    - 한 패키지만 다시 배포할 때는 같은 메뉴의 `Export 1. Basics` / `Export 2. Clicker`로 개별 익스포트할 수 있습니다.
3. **배포 전 점검 체크리스트**:
    - [ ] 강사용 런타임 코드(`Assets/Scripts/`)나 씬 구성을 고친 적이 있다면 대응 `My*` 스크립트에도 반영했는가 (5절 운영 규칙)
    - [ ] `Create All Student Scenes`로 학생용 씬을 재생성했는가
    - [ ] 코드 변경 후 Unity Console에 컴파일 에러가 없는가 (Unity MCP `Unity_GetConsoleLogs`)
    - [ ] (권장, 배포 전 1회) 별도 빈 URP 프로젝트에 4절 절차 그대로 임포트해 두 씬 Play 확인 — 특히 1부 패키지를 먼저 넣어 한글이 깨지지 않는지

> ℹ️ CLI/배치에서는 `StudentPackageCreator.BuildAndExportBatch()`를 호출하면 씬 생성 후 `STUDENT_PACKAGE_OUTPUT` 환경변수 폴더로 전체 익스포트합니다(씬 생성 실패 시 비정상 종료).

## 4. 학생: 프로젝트 셋업 절차 (수업 전 안내)

학생에게 아래 순서를 안내합니다(노션 교안에 같은 내용을 게시해 두면 좋습니다). 설치 절차 전체 설명은 [교안 02 §1](../교안/02.%20R3%20시작하기%20-%20Observable%20만들기.md)에 있습니다.

1. **빈 URP 프로젝트 생성**: Unity 6.3에서 **Universal 3D (URP) 템플릿**으로 새 프로젝트 생성.
    - ⚠️ Built-in 템플릿으로 만들면 머티리얼이 깨져(분홍색) 보입니다. 반드시 URP 템플릿을 사용하세요.
2. **R3 설치** — R3는 **두 부분**입니다(R3 본체는 범용 .NET 라이브러리라 NuGet으로, 유니티 통합은 git URL로).
    1. **NuGetForUnity 설치**: `Window > Package Manager` → `+` → _Install package from git URL..._ →
       `https://github.com/GlitchEnzo/NuGetForUnity.git?path=/src/NuGetForUnity`
    2. **R3 본체 설치**: 메뉴 `NuGet > Manage NuGet Packages`에서 `R3` 검색 → Install (`Microsoft.Bcl.TimeProvider` 등 의존 패키지 자동 설치).
    3. **R3.Unity 설치**: Package Manager → git URL →
       `https://github.com/Cysharp/R3.git?path=src/R3.Unity/Assets/R3.Unity#<버전>`
       (`#<버전>` 태그로 코어 NuGet 버전과 일치시켜 고정. 최신은 [R3 릴리스 페이지](https://github.com/Cysharp/R3/releases) 확인.)
    - 설치 확인: 아무 스크립트에서 `using R3;`가 컴파일되면 완료.
3. **UniTask 설치**: Package Manager → `+` → _Install package from git URL..._ →
   `https://github.com/Cysharp/UniTask.git?path=src/UniTask/Assets/Plugins/UniTask#2.5.10`
    - 1부 데모의 비동기 패널(`MyAsyncDemo`)이 단발 비동기를 UniTask로 작성합니다. 설치 확인: `using Cysharp.Threading.Tasks;`가 컴파일되면 완료. (최신 버전은 [UniTask 릴리스](https://github.com/Cysharp/UniTask/releases)에서 확인.)
4. **Input System 설치**: `Window > Package Manager` → Unity Registry에서 **Input System** 설치 → "Enable the new input backends?" 팝업에서 **Yes**(에디터 재시작). 데모 씬의 `EventSystem`은 `InputSystemUIInputModule`을 쓰므로 New 입력 핸들러가 필요합니다.
5. **패키지 임포트**: 배포받은 `.unitypackage`를 더블클릭(또는 `Assets > Import Package > Custom Package...`) → 전체 선택 → Import.
    - **`R3Lecture-Student-1-Basics`(1부)를 수업 시작 때 가장 먼저** 임포트하세요. 한글 폰트·TMP 설정이 이 패키지에 들어 있어, 건너뛰면 한글이 깨집니다.
    - 2부에 들어갈 때 `R3Lecture-Student-2-Clicker`를 임포트합니다.
6. **확인**: `Assets/StudentPackage/Scenes/ReactiveBasics` 를 열고 Play → 5개 섹션의 버튼이 동작하고 한글이 정상 표시되면 셋업 완료.
    - 단, **클리커(`ReactiveClicker`)는 스켈레톤**이라 Play해도 버튼이 무반응인 것이 정상입니다(2절). 수업에서 메서드를 채우면 동작합니다.

## 5. 운영·동기화 규칙

- **동기화**: 강사용 런타임 코드(`Assets/Scripts/`)나 씬 구성을 바꾸면 → 대응하는 `Assets/StudentPackage/Scripts/`의 `My*` 스크립트를 같이 고치고 → `Create All Student Scenes`로 씬을 재생성하고 → 해당 패키지를 다시 익스포트합니다.
- **필드명 유지**: `My*` 스크립트의 직렬화 필드 이름·접근제어자는 원본과 동일해야 합니다. 씬 생성 도구(`StudentPackageCreator`)가 필드명을 기준으로 Inspector 참조를 복사하며, 어긋나면 씬 생성이 명시적 예외로 실패합니다(조용히 깨지지 않음).
- **GUID 검증**: 익스포트 직전 각 에셋 경로의 GUID를 확인합니다. 학생용 씬을 재생성하지 않았거나 파일이 이동·개명되면 익스포트가 예외로 멈춥니다 — 1번(씬 생성) → 2번(익스포트) 순서를 지키세요.
- **씬이 깨졌을 때**: 강사 프로젝트는 `Create All Student Scenes`로 재생성, 학생 프로젝트는 패키지를 다시 임포트하면 초기 상태로 돌아갑니다(스크립트에 작성한 코드도 덮어써지니, 학생이 작성 중인 코드는 백업하도록 안내).
- **수업 중 학생이 따라오지 못할 때(2부)**: 모범답안은 강사 프로젝트의 `Assets/Scripts/ClickerGame/`에 있습니다. 화면 공유로 해당 메서드만 보여 주고 따라 입력하게 하세요.
