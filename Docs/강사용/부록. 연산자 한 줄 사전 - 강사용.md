## 생성 — 스트림을 만듦

| 연산자        | 한 줄                                           |
| ------------- | ----------------------------------------------- |
| `Range`       | 시작값부터 정수 N개 발행 후 완료(콜드)          |
| `Timer`       | 지정 시간 뒤 1발, 또는 첫 발 뒤 주기 반복(Unit) |
| `Interval`    | 일정 간격으로 Unit 무한 발행                    |
| `FromEvent`   | 기존 C# event를 스트림으로 감쌈                 |
| `EveryUpdate` | 매 프레임 Unit — 전역 스트림이라 수명관리 필수  |

## 필터 — 조건·개수로 걸러냄

| 연산자                 | 한 줄                                             |
| ---------------------- | ------------------------------------------------- |
| `Where`                | 조건을 통과한 값만                                |
| `Distinct`             | 지금까지 나온 적 없는 값만(전체 중복 제거)        |
| `DistinctUntilChanged` | 직전 값과 다를 때만(연속 중복 제거)               |
| `Take`                 | 처음 N개만 받고 완료(시간 오버로드는 지정 시간만) |
| `Skip`                 | 처음 N개(또는 지정 시간) 건너뛰고 이후만          |

## 변환 — 값을 다른 모양으로

| 연산자   | 한 줄                                          |
| -------- | ---------------------------------------------- |
| `Select` | 각 값을 다른 값·모양으로 변환(LINQ map)        |
| `Cast`   | 각 값을 지정 타입으로 캐스팅                   |
| `Index`  | 각 값에 0부터의 순번을 붙여 `(index, value)`로 |

## 시간 — 시간으로 다듬음

| 연산자          | 한 줄                                                 |
| --------------- | ----------------------------------------------------- |
| `Debounce`      | 잠잠해지면 마지막 값 1발(입력 끝). UniRx의 Throttle   |
| `ThrottleFirst` | 구간 첫 발만 통과, 나머지 무시(쿨다운)                |
| `ThrottleLast`  | 주기마다 구간 마지막 값(UniRx의 Sample)               |
| `Delay`         | 모든 발행을 지정 시간만큼 뒤로 미룸                   |
| `Chunk`         | 경계(개수·시간)까지 모아 배열 `T[]`로(UniRx의 Buffer) |

## 결합 — 여러 스트림을 합침

| 연산자           | 한 줄                                                          |
| ---------------- | -------------------------------------------------------------- |
| `Merge`          | 여러 스트림을 도착 순서대로 하나로 섞음                        |
| `CombineLatest`  | 한쪽이 바뀔 때마다 모든 스트림의 최신값 조합(버튼 활성화 정석) |
| `Zip`            | 각 스트림의 n번째끼리 짝지음(느린 쪽 기준)                     |
| `WithLatestFrom` | 주도 스트림이 발화하는 순간 다른 쪽 최신값을 샘플              |
| `Concat`         | 앞 스트림이 완료된 뒤 다음 스트림을 이어 붙임                  |

## 누적 — 흘러온 값을 누적함

| 연산자      | 한 줄                                                     |
| ----------- | --------------------------------------------------------- |
| `Scan`      | 누적하며 매 단계 중간 결과를 발행(달리는 합계)            |
| `Aggregate` | 완료될 때까지 누적해 최종값 1발만(`AggregateAsync`, Task) |

## 에러·완료 — 에러·완료를 다룸 (5차시)

| 연산자                   | 한 줄                                                                              |
| ------------------------ | ---------------------------------------------------------------------------------- |
| `OnErrorResume`          | R3 기본 에러 시그널 — 에러를 통지하되 스트림을 끝내지 않음                         |
| `Catch`                  | 에러로 끝나려 할 때 다른 Observable로 갈아타 복구                                  |
| `OnErrorResumeAsFailure` | `OnErrorResume`의 에러를 `OnCompleted(Failure)`로 바꿔 종료(전통 Rx의 OnError처럼) |

> R3는 전통 Rx와 달리 예외가 기본적으로 `OnErrorResume`으로 흐르고 **구독이 끊기지 않음**(`OnError → 종료`가 아님). 그래서 `Catch`는 `OnErrorResume`을 못 잡고 Failure 완료만 가로챔 — 통합 에러 처리는 `OnErrorResumeAsFailure`로 먼저 종료로 전환한 뒤 한다.

## 공유 — 하나의 구독을 여럿이 나눠 씀

| 연산자        | 한 줄                                                                       |
| ------------- | --------------------------------------------------------------------------- |
| `Share`       | 구독 하나를 여럿이 공유(소스 중복 실행 방지). `Publish().RefCount()`와 같음 |
| `Replay` 계열 | 과거 값을 버퍼에 담아 새 구독자에게 다시 흘려줌(`Connect` 필요)             |

---

> **이름 함정** — R3엔 `Throttle`·`Retry`가 **없음**(각각 `Debounce`, `Catch`·async 루프로 대체). `ThrottleLast`=UniRx `Sample`, `Chunk`=UniRx `Buffer`(`.Count`→`.Length`). 전체 매핑·시그니처·마블·조합 레시피는 [부록. 연산자·조합 레퍼런스](부록.%20연산자·조합%20레퍼런스%20-%20강사용.md).
