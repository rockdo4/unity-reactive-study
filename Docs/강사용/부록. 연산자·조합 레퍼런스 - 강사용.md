## 0. 읽는 법

**마블 표기** — 가로선은 시간(왼→오), 구슬은 발화한 값, `|`는 완료(OnCompleted), `-->`는 계속. 두 줄 이상은 같은 시간축에 세로로 정렬해 읽음.

```
in:    --a--b--c-->     입력 스트림
op:    --a-----c-->     연산 후
```

**시그니처 표기** — R3 1.3.1 기준. `this`가 붙은 건 확장 메서드(`source.Op(...)` 형태로 체인), `Observable.`로 시작하는 건 정적 팩토리. 강의에서 실제로 쓰는 오버로드만 싣고, 나머지(`TimeProvider`·`CancellationToken`·인덱스 버전 등)는 "그 외 오버로드" 줄로 요약. R3는 스케줄러를 인자로 받지 않고 `TimeProvider`/`FrameProvider`로 받는 게 .NET 표준과 다름.

> R3는 연산자 이름을 .NET LINQ/dotnet-reactive 표준에 맞춰 둠. UniRx 코드를 그대로 붙이면 컴파일 안 되는 함정(`Throttle→Debounce`, `Buffer→Chunk` 등)은 §3 매핑표.

## 1. 연산자 카탈로그

### 1.1 생성 — 스트림을 만든다

대응 데모: `CreateStreamsDemo.cs`(교안 02), `OperatorsDemo.cs`·`ClickerModel.cs`.

#### Observable.Range

한 줄: 구독하는 순간 `start`부터 `count`개를 동기로 쏟아내고 즉시 완료하는 **콜드** 스트림.

```csharp
public static Observable<int> Range(int start, int count)
// 그 외: Range(int start, int count, CancellationToken cancellationToken)
```

```
Range(1, 4):  (구독)→1-2-3-4-|     구독 즉시 전부 발행 후 완료
```

```csharp
// CreateStreamsDemo.cs
Observable.Range(1, 4)
    .Subscribe(x => Log($"Range OnNext: {x}"), result => Log($"Range OnCompleted: {result}"));
```

#### Observable.Timer

한 줄: `dueTime` 뒤 한 번 발행하고 완료(period 주면 그 뒤 주기 반복).

```csharp
public static Observable<Unit> Timer(TimeSpan dueTime, CancellationToken cancellationToken = default)
// 그 외: Timer(dueTime, period, ...) 주기 반복 / DateTimeOffset·TimeProvider 오버로드
```

```
Timer(2s):  ----------U|     2초 뒤 Unit 1발 후 완료
```

```csharp
// ClickerModel.cs — 피버 5초 뒤 자동 종료
Observable.Timer(FeverDuration).Subscribe(_ => IsFever.Value = false).AddTo(m_Disposables);
```

#### Observable.Interval

한 줄: `period`마다 무한 발행. 멈추려면 구독을 `Dispose`하거나 토큰 취소(수명 관리 필수).

```csharp
public static Observable<Unit> Interval(TimeSpan period, CancellationToken cancellationToken = default)
```

```
Interval(1s):  --U--U--U--U-->     1초마다 Unit, 끝없음
```

```csharp
// ClickerModel.cs — 1초마다 자동 수입(도우미 보유 시)
Observable.Interval(TimeSpan.FromSeconds(1))
    .Where(_ => GoldPerSecond.Value > 0)
    .Subscribe(_ => Gold.Value += GoldPerSecond.Value);
```

#### Observable.FromEvent

한 줄: 기존 C# `event`를 Observable로 감쌈. 구독 해제 시 `-=`까지 자동.

```csharp
public static Observable<T> FromEvent<T>(Action<Action<T>> addHandler, Action<Action<T>> removeHandler, CancellationToken cancellationToken = default)
// 그 외: 인자 없는 FromEvent(Action<Action> ...) / 변환자(conversion) 오버로드
```

```csharp
// CreateStreamsDemo.cs — event Action<int> LegacyScoreEvent 를 스트림으로
Observable.FromEvent<int>(h => LegacyScoreEvent += h, h => LegacyScoreEvent -= h)
    .Subscribe(score => Log($"FromEvent 수신: 점수 {score}"));
```

#### Observable.EveryUpdate (R3.Unity)

한 줄: 매 프레임 `Unit` 1발. 폴링형 입력·시간 로직의 출발점. **전역 스트림이라 `AddTo` 수명 관리 필수**.

```csharp
public static Observable<Unit> EveryUpdate()
// 그 외: EveryUpdate(CancellationToken) / FrameProvider 지정 오버로드
```

```
EveryUpdate():  -U-U-U-U-U-U-->     매 프레임 Unit
```

```csharp
// OperatorsDemo.cs — 매 프레임에서 "왼쪽 버튼 눌린 프레임"만 거름
Observable.EveryUpdate()
    .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame);
```

> 그 밖 팩토리: `Observable.Return(T value)` — 값 1개 발행 후 완료. `Observable.Empty`/`Observable.Never`도 있음.

### 1.2 발행기·상태

대응 데모: `CreateStreamsDemo.cs`, `BindingDemo.cs`, `ClickerModel.cs`, `TimingBarModel.cs`.

#### Subject\<T\>

한 줄: 코드 어디서든 `OnNext`로 값을 직접 미는 발행기(현재값은 안 들고 있음).

```csharp
public Subject()
public void OnNext(T value)
public void OnCompleted()           // Dispose 시 자동 OnCompleted
```

```csharp
// CreateStreamsDemo.cs
private readonly Subject<string> m_Subject = new();
m_Subject.Subscribe(msg => Log($"Subject 수신: {msg}"));
m_Subject.OnNext($"메시지 #{++m_SubjectCount}");
```

#### ReactiveProperty\<T\> / SerializableReactiveProperty\<T\>

한 줄: **현재값을 들고 있는** 발행기. 구독하면 즉시 현재값 1발, 이후 값이 바뀔 때마다 통지. 같은 값 대입은 통지 안 함.

```csharp
public ReactiveProperty(T initialValue = default)
public T Value { get; set; }        // 대입 → 구독자에게 자동 통지
// SerializableReactiveProperty<T> : ReactiveProperty<T> — [SerializeField] 직렬화 가능(인스펙터 편집)
public SerializableReactiveProperty(T value)
```

```
RP(100):  100 ─(구독 즉시)→ ─.Value=90→ ─.Value=90(무시)→ ─.Value=70→
구독자:    100              90                              70
```

```csharp
// BindingDemo.cs — 일반 ReactiveProperty는 [SerializeField] 불가라 Serializable 버전 사용
[SerializeField] private SerializableReactiveProperty<int> m_Hp = new(100);
m_Hp.Subscribe(hp => m_HpText.text = $"HP : {hp}");   // 구독 즉시 100이 한 번 흐름
m_Hp.Value = Mathf.Max(0, m_Hp.Value - 10);           // 대입이 곧 통지
```

#### ReadOnlyReactiveProperty\<T\> / ToReadOnlyReactiveProperty

한 줄: 읽기 전용(`Value` get만). 파생 상태를 외부에 노출할 때 — Model→Presenter 단방향 바인딩의 정석.

```csharp
public T Value { get; }
public static ReadOnlyReactiveProperty<T> ToReadOnlyReactiveProperty<T>(this Observable<T> source)
// 그 외: ToReadOnlyReactiveProperty(source, T initialValue)
```

```csharp
// ClickerModel.cs — "골드 ≥ 비용"을 자동 재계산되는 읽기 전용 상태로 노출
public ReadOnlyReactiveProperty<bool> CanBuyClickUpgrade { get; }
CanBuyClickUpgrade = Gold.CombineLatest(ClickUpgradeCost, (gold, cost) => gold >= cost)
    .ToReadOnlyReactiveProperty();
```

### 1.3 필터·변환 — LINQ와 같은 이름, 같은 의미

대응 데모: `OperatorsDemo.cs`, `BindingDemo.cs`, `TimingBarPresenter.cs`. 대상이 "시간 위의 스트림"일 뿐 LINQ 그대로.

#### Where

한 줄: 조건 통과한 값만 흘려보냄.

```csharp
public static Observable<T> Where<T>(this Observable<T> source, Func<T, bool> predicate)
// 그 외: 인덱스 버전 Where(source, Func<T, int, bool>)
```

```
in:    --(-1)--2--0--3-->
Where: -------2-----3-->     (x => x > 0)
```

#### Select

한 줄: 값을 다른 모양으로 변환.

```csharp
public static Observable<TResult> Select<T, TResult>(this Observable<T> source, Func<T, TResult> selector)
// 그 외: 인덱스 버전 Select(source, Func<T, int, TResult>)
```

```
in:     --1--2--3-->
Select: --10-20-30-->     (x => x * 10)
```

#### DistinctUntilChanged

한 줄: **연속**된 같은 값만 제거("바뀔 때만 한 번"). 떨어져 있는 같은 값은 통과.

```csharp
public static Observable<T> DistinctUntilChanged<T>(this Observable<T> source)
// 그 외: comparer / keySelector 오버로드
```

```
in:       --a--a--b--b--a-->
Distinct: --a-----b-----a-->     (연속 중복만 제거, 끝 a는 b 뒤라 통과)
```

```csharp
// BindingDemo.cs — HP를 구간 라벨로 바꾸고, 라벨이 바뀔 때만 UI 갱신
m_Hp.Select(hp => hp <= 0 ? "쓰러짐" : hp < 30 ? "위험" : hp < 70 ? "주의" : "안전")
    .DistinctUntilChanged()
    .Subscribe(state => m_HpStateText.text = $"상태: {state}");
```

#### Take / Skip

한 줄: `Take(n)`은 처음 n개만 받고 **완료**(OnCompleted). `Skip(n)`은 처음 n개를 버리고 그 뒤만.

```csharp
public static Observable<T> Take<T>(this Observable<T> source, int count)
public static Observable<T> Skip<T>(this Observable<T> source, int count)
// 그 외: TimeSpan 버전(시간 기준)
```

```
in:      --a--b--c--d-->
Take(2): --a--b|              (2개 받고 완료)
Skip(2): -------c--d-->       (처음 2개 버림)
```

```csharp
// TimingBarPresenter.cs — 정해진 횟수만 받고 라운드 종료(완료가 곧 라운드 끝 신호)
HitStream().WithLatestFrom(m_Phase, (_, phase) => phase).Take(m_AttemptsPerRound);
```

### 1.4 시간 — 상태 플래그·타이머 없이 시간 조건을 선언

대응 데모: `OperatorsDemo.cs`, `ClickerPresenter.cs`. 이 차시의 핵심.

#### Debounce

한 줄: 마지막 이벤트 후 `timeSpan` 동안 새 이벤트가 없으면 그 **마지막 값**을 내보냄("입력이 끝났을 때 한 번").

```csharp
public static Observable<T> Debounce<T>(this Observable<T> source, TimeSpan timeSpan)
// 그 외: TimeProvider 오버로드. (UniRx의 Throttle = R3의 Debounce — 최대 함정)
```

```
in:           --a-b--c----------d----->
Debounce(.5s):------------c----------d-->     (연타 중엔 침묵, 잠잠해지면 마지막)
```

```csharp
// OperatorsDemo.cs — 타이핑이 0.5초 멈췄을 때만, 직전과 다른 검색어일 때만 검색
m_SearchInput.onValueChanged.AsObservable()
    .Debounce(TimeSpan.FromMilliseconds(500))
    .DistinctUntilChanged()
    .Subscribe(query => Search(query));
```

#### ThrottleFirst

한 줄: **첫 이벤트를 통과**시키고 그 뒤 `timeSpan` 동안 나머지를 무시("먼저 발동하고 잠시 귀 닫기"). 쿨다운.

```csharp
public static Observable<T> ThrottleFirst<T>(this Observable<T> source, TimeSpan timeSpan)
// 그 외: TimeProvider / Observable<TSample> 경계 오버로드
```

```
in:           --a-b-c----d-e-------f-->
ThrottleFirst:--a--------d---------f-->     (1s: 발동 직후 구간은 무시)
```

```csharp
// ClickerPresenter.cs — 피버 버튼: 첫 클릭만 통과, 15초 쿨다운
m_FeverButton.OnClickAsObservable()
    .ThrottleFirst(FeverCooldown)
    .Subscribe(_ => { if (m_Model.TryStartFever()) Log("피버 시작!"); });
```

> R3에서 `ThrottleFirst`의 타이머는 **구독 시점이 아니라 값이 도착할 때** 시작함. 그래서 첫 값은 항상 통과.

#### ThrottleLast

한 줄: 고정 주기마다 그 구간의 **마지막 값**을 내보냄(UniRx의 `Sample`). 잠잠함과 무관하게 주기적으로 나옴 — Debounce와 대비점.

```csharp
public static Observable<T> ThrottleLast<T>(this Observable<T> source, TimeSpan timeSpan)
```

```
주기:         |---1s---|---1s---|
in:           --a--b------c--d----->
ThrottleLast: --------b--------d-->     (각 주기 경계에서 직전 구간의 마지막)
```

#### Chunk

한 줄: 이벤트를 모았다가 **경계**(시간·개수·다른 스트림)가 오면 **배열로** 묶어 내보냄. 반환이 `T[]`라 `.Length`로 셈(UniRx `Buffer`의 `IList.Count` 아님).

```csharp
public static Observable<T[]> Chunk<T>(this Observable<T> source, TimeSpan timeSpan, int count)
public static Observable<TSource[]> Chunk<TSource, TWindowBoundary>(this Observable<TSource> source, Observable<TWindowBoundary> windowBoundaries)
// 그 외: Chunk(count) / Chunk(count, skip) / Chunk(timeSpan) / 비동기 window 오버로드
```

```
// (a) 시간·개수 경계 — Chunk(.5s, 3)
in:          --a-b-c----d-e------->
Chunk:       ------[a,b,c]----[d,e]->     (3개 차거나 0.5초 경과 시 방출)

// (b) 다른 스트림을 경계로 — Chunk(boundary)
in:          --a-b--c--------d-e----->
boundary:    --------x----------x---->
Chunk:       --------[a,b,c]----[d,e]->
```

```csharp
// OperatorsDemo.cs — (a) 500ms 안에 3번 누르면 연타
taps.Chunk(TimeSpan.FromMilliseconds(500), 3)
    .Where(xs => xs.Length >= 3)
    .Subscribe(_ => SetText(m_TapText, "3연타!"));
```

> 더블클릭은 (b) 경계 버전 — §2 R2 레시피.

#### Delay

한 줄: 각 값을 `dueTime`만큼 통째로 늦춰 흘려보냄(값은 그대로, 타이밍만 이동).

```csharp
public static Observable<T> Delay<T>(this Observable<T> source, TimeSpan dueTime)
```

```

Delay(t):  ----a--b--c-->
```

### 1.5 결합 — 여러 스트림을 합친다

대응 데모: `OperatorsDemo`(이론), `BindingDemo.cs`, `ClickerModel.cs`, `TimingBarPresenter.cs`.

#### Merge

한 줄: 여러 스트림을 **도착 순서대로** 한 스트림에 합침("여러 입력원 → 한 행동"). 타입이 같아야 함.

```csharp
public static Observable<T> Merge<T>(this Observable<T> source, Observable<T> second)
// 그 외: Merge(params Observable<T>[]) / Merge(IEnumerable<Observable<T>>)
```

```
A:     --a-----c----->
B:     ----b-----d--->
Merge: --a-b---c-d--->
```

```csharp
// TimingBarPresenter.cs — HIT 버튼과 Space 키를 한 입력 스트림으로
m_HitButton.OnClickAsObservable()
    .Merge(Observable.EveryUpdate()
        .Where(_ => Keyboard.current != null && Keyboard.current.spaceKey.wasPressedThisFrame));
```

#### CombineLatest

한 줄: 어느 쪽이든 새 값이 오면 **모든 스트림의 최신값**을 묶어 내보냄. **모두 한 번씩 발화한 뒤**부터 흐름("조건이 여러 값에 걸칠 때").

```csharp
public static Observable<TResult> CombineLatest<T1, T2, TResult>(this Observable<T1> source1, Observable<T2> source2, Func<T1, T2, TResult> resultSelector)
// 그 외: 최대 15개 소스 / params Observable<T>[] → Observable<T[]>
```

```
HP:            --100--------70--------------30----->
MP:            --50-------------------10----------->
CombineLatest: --(100,50)---(70,50)---(70,10)-(30,10)->
```

```csharp
// BindingDemo.cs — HP·MP 둘 다 조건을 만족할 때만 스킬 버튼 활성화
m_Hp.CombineLatest(m_Mp, (hp, mp) => hp > 0 && mp >= SkillMpCost)
    .DistinctUntilChanged()
    .Subscribe(canUse => m_SkillButton.interactable = canUse);
```

#### WithLatestFrom

한 줄: **첫(주도) 스트림이 발화한 순간**에만, 둘째 스트림의 최신값을 샘플해 결합. 둘째가 주도하면 아무것도 안 나옴.

```csharp
public static Observable<TResult> WithLatestFrom<TFirst, TSecond, TResult>(this Observable<TFirst> first, Observable<TSecond> second, Func<TFirst, TSecond, TResult> resultSelector)
```

```
phase: -a-b-c-d-e-f-g-h->     매 프레임 갱신(늘 최신값 보유)
hit:   ---X-------X----->     주도 스트림
out:   ---b-------f----->     X 누른 순간의 최신 phase만 샘플
```

```csharp
// TimingBarPresenter.cs — 누른 '순간'의 마커 위치를 샘플
HitStream().WithLatestFrom(m_Phase, (_, phase) => phase);
```

> `CombineLatest`는 **양쪽 변화 모두** 반응, `WithLatestFrom`은 **한쪽이 주도**하고 다른 쪽은 참조만 — 입력 순간의 상태 샘플링에 정확히 맞음.

#### Zip

한 줄: 각 스트림의 **n번째 값끼리 짝**지어 내보냄(둘 다 n번째가 와야). 인덱스 매칭.

```csharp
public static Observable<TResult> Zip<T1, T2, TResult>(this Observable<T1> source1, Observable<T2> source2, Func<T1, T2, TResult> resultSelector)
// 그 외: 다중 소스 → Observable<T[]>
```

```
A:    --a-----b--------c-->
B:    ----1------2--3----->
Zip:  ----(a,1)--(b,2)-(c,3)->     (짝이 맞을 때만)
```

> 강의에서 `Zip`은 카탈로그 행(이름만). 게임 UI는 대개 `CombineLatest`로 충분.

### 1.6 누적 — 흘러온 값을 쌓는다

#### Scan

한 줄: `seed`에서 시작해 값마다 누적 결과를 **매번** 흘려보냄(LINQ `Aggregate`의 실시간 버전).

```csharp
public static Observable<TAccumulate> Scan<T, TAccumulate>(this Observable<T> source, TAccumulate seed, Func<TAccumulate, T, TAccumulate> accumulator)
// 그 외: seed 없는 Scan(source, Func<T,T,T>) — 첫 값을 seed로
```

```
in:   --5---3---10-->
Scan: --5---8---18-->     (seed 0, (누적, 새값) => 누적 + 새값)
```

```csharp
// 이론 03 — 획득 골드 → 누적 합계
goldGained.Scan(0L, (total, gained) => total + gained)
    .Subscribe(total => m_TotalText.text = $"누적 {total}");
```

> `Aggregate`도 시그니처는 같으나 **완료 시 최종값 1개만** 내보냄. 과정을 보려면 `Scan`, 결과만 필요하면 `Aggregate`.

### 1.7 공유

#### Share

한 줄: 콜드 원본을 **한 번만 구독**해 여러 구독자가 그 결과를 나눠 받게 함(중복 구독·중복 실행 방지).

```csharp
public static Observable<T> Share<T>(this Observable<T> source)
```

```csharp
// OperatorsDemo.cs — clicks를 본체와 Debounce 경계 두 곳이 씀 → Share로 구독 하나로 공유
var clickStream = Observable.EveryUpdate()
    .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame)
    .Share();
```

> 한 스트림(특히 `EveryUpdate`·네트워크 등 값비싼 원본)을 **두 곳 이상에서 구독**하면 콜드 재실행 의심 → `Share()` 검토. 더블클릭·콤보 레시피(§2)가 전형.

### 1.8 비동기·수명

대응 데모: `AsyncDemo.cs`, `TimingBarPresenter.cs`, `LifetimeDemo.cs`, `ClickerModel.cs`.

#### SubscribeAwait + AwaitOperation

한 줄: 각 값마다 비동기 작업(`ValueTask`)을 실행하되, **겹쳐 들어온 값을 어떻게 다룰지**를 `AwaitOperation`으로 선언. 토큰은 구독 해제 시 자동 취소.

```csharp
public static IDisposable SubscribeAwait<T>(
    this Observable<T> source,
    Func<T, CancellationToken, ValueTask> onNextAsync,
    AwaitOperation awaitOperation = AwaitOperation.Sequential,
    bool configureAwait = true, bool cancelOnCompleted = true, int maxConcurrent = -1)
// 그 외: onCompleted(Action<Result>) / onErrorResume 추가 오버로드
```

```csharp
public enum AwaitOperation
{
    Sequential = 0,        // 줄 세워 순서대로(이전 완료까지 대기)
    Drop = 1,              // 처리 중 들어온 값은 버림(연타 방지)
    Switch = 2,            // 이전 작업 취소하고 새 값으로 전환
    Parallel = 3,          // 즉시 병렬 처리
    SequentialParallel = 4,// 병렬 처리하되 결과만 순서대로
    ThrottleFirstLast = 5  // 처리 중엔 첫 값과 마지막 값만
}
```

```
click:      --c1--c2-c3-------c4-->     (각 클릭마다 1.5초 작업)
Sequential: --[c1….][c2….][c3….][c4….]->   전부, 줄 세워 순서대로
Drop:       --[c1….]----------[c4….]->     처리 중 c2·c3 버림
```

```csharp
// AsyncDemo.cs — 저장 버튼 연타 방지(처리 중 클릭 무시)
m_DropButton.OnClickAsObservable()
    .SubscribeAwait(async (_, ct) => { await FakeLoadAsync(ct); }, AwaitOperation.Drop);
```

> 단발 비동기 본체는 UniTask로 작성하고 이 `ValueTask` 람다 안에서 `await`함(`await UniTask.Delay(..., cancellationToken: ct)`). 프로젝트 규약.

#### WaitAsync

한 줄: 스트림이 **완료(OnCompleted)될 때까지** await. 값은 버리고 완료 신호만 기다림.

```csharp
public static Task WaitAsync<T>(this Observable<T> source, CancellationToken cancellationToken = default)
```

```csharp
// TimingBarPresenter.cs — Take(N)로 N회 뒤 완료되는 스트림을 끝까지 대기
Observable<float> roundHits = HitStream().WithLatestFrom(m_Phase, (_, p) => p).Take(N).Share();
await roundHits.WaitAsync(ct);   // N회 시도가 끝나면 다음 줄로
```

#### Subscribe

한 줄: 스트림에 관찰자를 등록(여기서 물이 흐르기 시작). `IDisposable` 반환 → 그게 곧 해제 핸들.

```csharp
public static IDisposable Subscribe<T>(this Observable<T> source, Action<T> onNext)
// 그 외: Subscribe(onNext, Action<Result> onCompleted) / onErrorResume 포함 오버로드
```

#### AddTo / CompositeDisposable

한 줄: 구독(`IDisposable`)의 수명을 오브젝트나 그릇에 묶음. `AddTo(this)`면 컴포넌트 파괴 시, `CompositeDisposable`면 그릇 `Dispose` 시 일괄 해제.

```csharp
public static T AddTo<T>(this T disposable, Component gameObjectComponent) where T : IDisposable
public static T AddTo<T>(this T disposable, GameObject gameObject) where T : IDisposable
public static T AddTo<T>(this T disposable, ICollection<IDisposable> collection) where T : IDisposable
```

```csharp
// LifetimeDemo.cs — 여러 구독을 한 그릇에, 한 번에 정리
private readonly CompositeDisposable m_Disposables = new();
Observable.Interval(TimeSpan.FromSeconds(5)).Subscribe(...).AddTo(m_Disposables);
m_Disposables.Clear();   // 담아 둔 구독 전부 해제(그릇은 재사용)
```

> `EveryUpdate`·`Interval`·`Timer` 같은 전역/타이머 스트림 구독은 `AddTo` 없으면 누수. `LifetimeDemo`의 "누수 만들기" 버튼 + `Window > Observable Tracker`로 시연.

#### OnClickAsObservable / AsObservable (R3.Unity)

한 줄: Unity UI 이벤트를 Observable로. `OnClickAsObservable`은 버튼 클릭을, `AsObservable`은 임의의 `UnityEvent`(예: `onValueChanged`)를 스트림으로.

```csharp
public static Observable<Unit> OnClickAsObservable(this Button button)
public static Observable<Unit> AsObservable(this UnityEvent unityEvent, CancellationToken cancellationToken = default)
public static Observable<T> AsObservable<T>(this UnityEvent<T> unityEvent, CancellationToken cancellationToken = default)
```

## 2. 조합 상용 예 (레시피)

데모에서 실제로 나오는 조합. 각 레시피는 **의도 한 줄 → 코드 → 동작 → 출처**.

### R1. 입력이 끝났을 때 한 번 — `Debounce` + `DistinctUntilChanged`

의도: 검색창 자동완성·설정 자동 저장. 타이핑이 멈췄고, 직전과 다를 때만.

```csharp
input.onValueChanged.AsObservable()
    .Debounce(TimeSpan.FromMilliseconds(500))   // 0.5초 잠잠 → 마지막 값 1발
    .DistinctUntilChanged()                     // 직전 검색어와 같으면 무시
    .Subscribe(query => Search(query));
```

동작: 연타 중 침묵 → 멈추면 마지막 값 → 그게 직전과 다를 때만 통과. 출처: `OperatorsDemo.cs`.

### R2. 더블클릭 — `Share` + `Chunk(Debounce 경계)` + `Where`

의도: 코루틴·플래그·타이머 없이 "클릭이 잠잠해질 때까지 모아서, 2개 이상이면 더블클릭".

```csharp
var clicks = Observable.EveryUpdate()
    .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame)
    .Share();                                            // 본체·경계 두 곳이 공유

clicks.Chunk(clicks.Debounce(TimeSpan.FromMilliseconds(250)))  // 경계 = 잠잠해진 순간
    .Where(clicks => clicks.Length >= 2)
    .Subscribe(_ => SetText(m_DoubleClickText, "더블클릭!"));
```

```
click:          --c-c--------c--c--c--------->
경계(Debounce): -------x----------------x---->     (250ms 잠잠)
Chunk:          -------[c,c]------------[c,c,c]->
Where(≥2):      -------더블!-----------더블!--->
```

출처: `OperatorsDemo.cs`. `Share`가 빠지면 본체와 경계가 `EveryUpdate`를 각자 구독해 두 벌 돎.

### R3. N연타 감지 — `Chunk(TimeSpan, count)` + `Where`

의도: 0.5초 안에 3번 누르면 연타.

```csharp
taps.Chunk(TimeSpan.FromMilliseconds(500), 3)   // 3개 차거나 0.5초 경과 시 묶어 방출
    .Where(xs => xs.Length >= 3)
    .Subscribe(_ => SetText(m_TapText, "3연타!"));
```

출처: `OperatorsDemo.cs`. R2와 차이: 경계가 **시간+개수**(R2는 다른 스트림).

### R4. 쿨다운 게이트 — `ThrottleFirst`

의도: 스킬/피버 버튼. 첫 입력만 발동, 일정 시간 나머지 무시.

```csharp
m_FeverButton.OnClickAsObservable()
    .ThrottleFirst(FeverCooldown)               // 첫 클릭 통과, 15초 무시
    .Subscribe(_ => { if (m_Model.TryStartFever()) Log("피버 시작!"); });
```

출처: `ClickerPresenter.cs`. `Time.time - last > cd` 수동 비교를 한 줄로.

### R5. 여러 입력원 → 한 행동 — `Merge`(+ `ThrottleFirst`)

의도: 키보드든 버튼이든 같은 행동. 합친 뒤 공통 쿨다운.

```csharp
fireByKey.Merge(fireByButton)                   // 둘 중 무엇이 와도 한 스트림
    .ThrottleFirst(TimeSpan.FromSeconds(0.5))   // 합친 뒤 공통 쿨다운
    .Subscribe(_ => Fire());
```

출처: 이론 03 / `TimingBarPresenter.HitStream()`(HIT 버튼 + Space). `Merge`는 타입이 같아야 하므로 한쪽을 `Select(_ => Unit.Default)`로 맞추기도.

### R6. 버튼 활성화의 정석 — `CombineLatest` + `DistinctUntilChanged`

의도: 여러 값에 걸친 조건이 바뀔 때만 버튼 상태 갱신.

```csharp
hp.CombineLatest(mp, (h, m) => h > 0 && m >= 20)  // 어느 쪽이 바뀌든 둘의 최신값으로 평가
    .DistinctUntilChanged()                       // 결과(true/false) 바뀔 때만
    .Subscribe(canUse => m_SkillButton.interactable = canUse);
```

출처: `BindingDemo.cs` / `ClickerModel.cs`(`Gold` × `Cost` → `CanBuy*`). 대부분의 게임 UI 조건이 이 형태.

### R7. 콤보 누적기 — `Share` + `Select` + `Merge`(Debounce 리셋) + `Scan`

의도: 클릭하면 +1, 0.8초 잠잠하면 0으로 리셋되는 콤보 카운터.

```csharp
var combo = clicks
    .Select(_ => 1)                                       // 클릭 → +1 신호
    .Merge(clicks.Debounce(ComboResetDelay).Select(_ => 0)) // 잠잠 → 0(리셋) 신호
    .Scan(0, (acc, x) => x == 0 ? 0 : acc + 1)            // 0이면 리셋, 아니면 누적 +1
    .Share();
```

```
click:    --c--c--c----------c-->
+1:       --1--1--1----------1-->
리셋:     -----------0--------->     (0.8초 잠잠)
Merge:    --1--1--1--0-------1-->
Scan:     --1--2--3--0-------1-->
```

출처: `ClickerPresenter.cs`. 시간 연산자(Debounce)·결합(Merge)·누적(Scan)이 한 파이프라인에 모인 종합 예.

### R8. 실시간 누적 합계 — `Scan`

의도: 흘러온 값의 누적을 매번 표시.

```csharp
goldGained.Scan(0L, (total, gained) => total + gained)
    .Subscribe(total => m_TotalText.text = $"누적 {total}");
```

출처: 이론 03. R7 콤보의 단순형.

### R9. 입력 순간의 상태 샘플 — `WithLatestFrom` (+ `Take` + `Share` + `WaitAsync`)

의도: 버튼을 누른 그 프레임의 마커 위치를 집어 판정. 정해진 횟수 뒤 라운드 종료.

```csharp
Observable<float> roundHits = HitStream()
    .WithLatestFrom(m_Phase, (_, phase) => phase)   // 누른 순간의 위상 샘플
    .Take(m_AttemptsPerRound)                       // N회 받고 완료
    .Share();                                       // 처리 구독 + 완료 대기가 공유

roundHits.Subscribe(phase => FlashJudgement(m_Model.ApplyHit(phase)));
await roundHits.WaitAsync(ct);                       // N회 끝나면(완료) 다음으로
```

출처: `TimingBarPresenter.cs`. `m_Phase`는 매 프레임 갱신되는 Hot 값(`ReactiveProperty<float>`).

### R10. 재진입 방지 — `SubscribeAwait(Drop)`

## 3. 빠른 참조

### 한 줄 요약

| 연산자                         | 한 줄                                         |
| ------------------------------ | --------------------------------------------- |
| `Range` / `Timer` / `Interval` | 콜드 N발·완료 / 1발 후 완료 / 주기 무한       |
| `FromEvent` / `EveryUpdate`    | C# event 감쌈 / 매 프레임 Unit(수명관리 필수) |
| `Subject` / `ReactiveProperty` | 직접 미는 발행기 / 현재값 보유+자동통지       |
| `Where` / `Select`             | 조건 통과만 / 값 변환                         |
| `DistinctUntilChanged`         | 연속 중복 제거(바뀔 때만)                     |
| `Take` / `Skip`                | 처음 n개만(후 완료) / 처음 n개 버림           |
| `Debounce`                     | 잠잠해지면 마지막 값(입력 끝)                 |
| `ThrottleFirst`                | 첫 발 통과, 구간 무시(쿨다운)                 |
| `ThrottleLast`                 | 주기마다 구간 마지막(Sample)                  |
| `Chunk`                        | 경계까지 모아 배열(`T[]`, `.Length`)          |
| `Merge`                        | 여러 스트림 도착순 합침                       |
| `CombineLatest`                | 모든 스트림 최신값 조합(버튼 활성화 정석)     |
| `WithLatestFrom`               | 주도 스트림 발화 순간 다른 쪽 최신값 샘플     |
| `Zip`                          | n번째끼리 짝                                  |
| `Scan` / `Aggregate`           | 누적 매번 / 완료 시 최종 1발                  |
| `Share`                        | 구독 하나를 여럿이 공유(중복실행 방지)        |
| `SubscribeAwait`               | 값마다 비동기 + 겹침 정책(Sequential/Drop/…)  |
| `WaitAsync`                    | 완료까지 await                                |
| `AddTo`                        | 구독 수명을 오브젝트/그릇에 묶음              |

### 어떤 시간/결합 연산자를 고르나

- "입력이 **끝났을 때** 한 번" → `Debounce`
- "**먼저** 한 번, 그 뒤 무시" → `ThrottleFirst`
- "**주기적으로** 최신값" → `ThrottleLast`
- "경계까지 **모아서**" → `Chunk`
- "여러 입력원 → **한 행동**" → `Merge`
- "여러 값에 걸친 **조건**" → `CombineLatest`
- "입력 **순간의 상태** 샘플" → `WithLatestFrom`

### UniRx → R3 이름 매핑 (옛 코드 읽을 때)

R3는 이름을 .NET 표준에 맞춰 바꿈. UniRx 코드를 그대로 붙이면 컴파일 안 됨.

| UniRx                           | R3                            | 함정                                         |
| ------------------------------- | ----------------------------- | -------------------------------------------- |
| `Throttle`                      | `Debounce`                    | **최대 함정** — R3엔 `Throttle` 없음         |
| `Buffer`                        | `Chunk`                       | 반환 `IList<T>`(`.Count`) → `T[]`(`.Length`) |
| `Sample`                        | `ThrottleLast`                |                                              |
| `StartWith`                     | `Prepend`                     |                                              |
| `ThrottleFirst`                 | `ThrottleFirst`               | 이름·동작 그대로                             |
| `IObservable<T>`/`IObserver<T>` | `Observable<T>`/`Observer<T>` | 인터페이스 → 추상 클래스                     |
| `this.UpdateAsObservable()`     | `Observable.EveryUpdate()`    | 전역 스트림 수명관리 필수                    |

## 4. 검증·출처 노트

- 모든 시그니처는 **R3 1.3.1** 기준 — `Assets/Packages/R3.1.3.1/README.md`(버전 일치 공식 문서)의 연산자 표와 `Library/PackageCache/com.cysharp.r3@.../Runtime`(Unity 통합), 그리고 데모 `.cs`에 대조함. R3 버전을 올리면 이 표를 다시 맞출 것.
- 강의에서 안 쓰는 오버로드(`TimeProvider`/`FrameProvider`/`CancellationToken`/인덱스/다중 소스)는 "그 외 오버로드" 줄로 요약만 함 — 필요 시 README 표 참조.
- 가드레일(단정 금지): R3엔 `Throttle`·`Retry`가 **없음**(재시도는 `FromAsync` 루프나 `Catch` 조합). "극적 단순화" 같은 과장 대신 "더 짧고 선언적"까지만. 자세한 건 [강사 사전 학습 가이드 §3](00.%20강사%20사전%20학습%20가이드.md).
- 코드 조각은 가독성을 위해 `.AddTo(this)`·로그 등 일부를 생략한 곳이 있음. 완전한 형태는 출처 `.cs` 참조.
