## 데모 1: 더블클릭 — Chunk + Debounce 경계 (7분)

> 🔧 **오늘 쓸 부품**: 03차시 `EveryUpdate`·`Where`·`Share`·`Chunk`·`Debounce`.

빈 `SetupDoubleClick()`을 함께 채웁니다. `EveryUpdate → Where → Share`로 클릭 스트림을 만들고, `Chunk(Debounce 경계)`로 묶습니다:

```csharp
private void SetupDoubleClick()
{
    // 매 프레임 스트림에서 "왼쪽 버튼이 눌린 프레임"만 거른다 (Input System 사용)
    // Share(): 아래에서 본체와 Debounce 경계 두 곳이 같은 스트림을 쓰므로 구독을 하나로 공유한다
    var clickStream = Observable
        .EveryUpdate()
        .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame)
        .Share();

    // 클릭이 250ms 동안 잠잠해지면(Debounce) 그때까지 모인(Chunk) 클릭을 한 묶음으로.
    // 묶음에 2개 이상 들어 있으면 더블클릭이다.
    clickStream
        .Chunk(clickStream.Debounce(TimeSpan.FromMilliseconds(250)))
        .Where(clicks => clicks.Length >= 2)
        .Subscribe(clicks =>
            SetText(m_DoubleClickText, $"더블클릭! ({clicks.Length}회 묶음, 프레임 {Time.frameCount})")
        )
        .AddTo(this);
}
```

- **"한 문장으로 읽어 보세요"**: "클릭이 250ms 잠잠해질 때까지 모아서, 2개 이상이면 더블클릭." 1차시 명령형 더블클릭(코루틴+플래그+타이머)과 대비 — 의도가 그대로 코드.

> ⚠️ **함정**: `Share()` 질문이 나오면 — "clickStream을 본체와 Debounce 경계 두 곳이 써서, 구독을 하나로 공유하는 것"까지만. 깊은 핫/콜드 강의는 이론 3-5 수준으로 제한.

**실행·확인** (작성 후 Play):

1. 화면을 **천천히 두 번** 클릭 → 반응 없음.
2. 화면을 **빠르게 두 번** 클릭 → "더블클릭!" + 묶음 개수·프레임 번호. Debounce 경계(250ms)의 의미가 체감된다.

**이론 연결**: 3-3(Chunk·Debounce), 3-5(Share).

## 데모 2: 검색어 입력 — Debounce + DistinctUntilChanged (5분)

> 🔧 **오늘 쓸 부품**: 03차시 `Debounce`·`DistinctUntilChanged`.

빈 `SetupSearchDebounce()`를 함께 채웁니다 — 자동완성·설정 자동 저장에서 흔한 조합:

```csharp
private void SetupSearchDebounce()
{
    // 타이핑이 500ms 멈췄을 때만, 그리고 직전 검색어와 다를 때만 "검색"을 실행한다.
    // 자동완성·설정 자동 저장에서 흔히 쓰는 조합 (Debounce + DistinctUntilChanged)
    m_SearchInput
        .onValueChanged.AsObservable()
        .Debounce(TimeSpan.FromMilliseconds(500))
        .DistinctUntilChanged()
        .Subscribe(query =>
            SetText(
                m_SearchResultText,
                string.IsNullOrWhiteSpace(query) ? "검색 대기 중..." : $"검색 실행: \"{query}\""
            )
        )
        .AddTo(this);
}
```

- Debounce는 "**입력이 끝났을 때** 한 번", DistinctUntilChanged는 "직전과 같으면 무시"라는 두 역할을 또렷이 분리해 짚는다.

**실행·확인** (작성 후 Play):

1. 검색창에 타이핑하는 **중에는 침묵** → 멈추면 0.5초 뒤 "검색 실행".
2. 같은 검색어를 다시 입력 → **무시**(DistinctUntilChanged).

> 📌 이 `Debounce` 한 줄을 **방금 학생이 자기 스켈레톤에 막 작성**했습니다. 잠시 뒤 "고의 실패 시연"에서 바로 이 줄을 UniRx식 `Throttle`로 바꿔 보며 이름 매핑을 각인시킵니다.

**이론 연결**: 3-2(DistinctUntilChanged), 3-3(Debounce).

## 데모 3·4: 3연타와 쿨다운 — 한 버튼의 두 정책 (7분)

> 🔧 **오늘 쓸 부품**: 03차시 `Share`·`Chunk`·`ThrottleFirst`.

빈 `SetupTapAndCooldown()`을 함께 채웁니다 — **같은 `taps` 스트림에 두 파이프라인**이 독립적으로 붙습니다. **마블을 먼저, 코드를 나중에.** 칠판에 세 줄을 그리고 나서 코드를 칩니다:

```
taps:         --t-t-t--------t-->
ThrottleFirst:--t------------t-->   (먼저 한 번, 구간 무시)
Chunk(500,3): --[t t t]---------->   (500ms 안에 3개면 묶음 발화)
```

```csharp
private void SetupTapAndCooldown()
{
    var taps = m_TapButton.OnClickAsObservable().Share();

    // 500ms 안에 3번 누르면 연타 (Chunk: 시간 또는 개수 경계로 묶기)
    taps.Chunk(TimeSpan.FromMilliseconds(500), 3)
        .Where(xs => xs.Length >= 3)
        .Subscribe(_ => SetText(m_TapText, $"3연타! (프레임 {Time.frameCount})"))
        .AddTo(this);

    // 첫 클릭만 통과시키고 2초간 무시 (ThrottleFirst: 스킬 쿨다운)
    taps.ThrottleFirst(TimeSpan.FromSeconds(2))
        .Subscribe(_ => SetText(m_CooldownText, $"발동! 2초 쿨다운 (프레임 {Time.frameCount})"))
        .AddTo(this);
}
```

- 강조점: "**스트림 하나에 정책 여러 개**". 여기서 `Share()`가 한 버튼 클릭을 두 파이프라인이 나눠 받게 한다.

**실행·확인** (작성 후 Play):

1. 탭 버튼을 **빠르게 3번**(500ms 안에) → "3연타!". 느리게 누르면 묶음이 3개를 못 채워 무반응.
2. 탭 버튼을 **연타** → "발동!"은 2초에 한 번만. (같은 버튼인데 위 3연타와 독립적으로 동작.)

> 📌 **던질 질문**: "`Share()`를 빼면 어떻게 될까요?" — 동작은 하지만 버튼 클릭 구독이 두 벌이 됩니다. "두 곳 이상에서 쓰면 Share"(3차시) 회수 기회.

**이론 연결**: 3-3(Chunk·ThrottleFirst), 3-5(Share).

## 🔻 고의 실패 시연 — UniRx `Throttle` (3분, 효과 최고)

학생이 인터넷에서 검색해 오는 코드는 UniRx 기준일 확률이 높다. **방금 데모 2에서 직접 작성한 `Debounce` 줄**을 UniRx식 `Throttle`로 바꿔 보게 해 **컴파일 에러**를 Console에서 보여 주고 매핑표로 복구한다.

1. 학생이 `SetupSearchDebounce`에 막 작성한 `Debounce`를 UniRx식 `Throttle`로 바꿔 입력 → 저장.
    ```csharp
    .Throttle(TimeSpan.FromMilliseconds(500))   // ← UniRx 이름. R3엔 없음
    ```
2. Unity Console에 **컴파일 에러**(`Throttle` 정의 없음)가 뜬다. 이때 Unity MCP `Unity_GetConsoleLogs`로도 확인 가능.
3. 이론 3-5의 이름 매핑표로 복구 — 학생은 다시 `Debounce`로 되돌린다: `Throttle → Debounce`(**최대 함정** — R3엔 Throttle이 아예 없음), `Buffer → Chunk`(`.Count → .Length`).
4. **멘트**: "백 마디 설명보다 컴파일 에러 한 번이 매핑을 각인시킨다."

> 🔧 학생 코드가 꼬이면 `R3Lecture-Student-1-Basics` 패키지 재임포트로 스켈레톤을 초기화(작성 중 코드는 백업 안내). 강사 데모 코드가 꼬이면 `git checkout -- Assets/Scripts/`로 원복한 뒤 다시 Play.

## 마무리 Q&A 예상 질문

> ❓ **체인이 길어지면 디버깅은 어떻게 하나요?**
> ①중간 스트림을 지역 변수로 쪼개 이름을 붙인다 ②`Do`(R3의 사이드 이펙트 연산자)나 중간 `Subscribe`+로그로 단계별 값을 찍는다 ③Tracker로 구독 자체를 확인(4차시). "7~8개 이상 이어지면 설계를 의심"이 교안의 권고.

> ❓ **UniRx 코드의 `Buffer`·`Throttle`을 R3로 옮길 때?** > `Buffer → Chunk`(결과 `.Count → .Length`), `Throttle → Debounce`(R3에 Throttle 없음). 방금 본 고의 실패 시연 그대로.

## 이해도 퀴즈

1. 같은 연타 입력에 대해 Debounce와 ThrottleFirst가 각각 언제 발화하는지 마블로 그리시오.
2. 더블클릭 파이프라인 `clicks.Chunk(clicks.Debounce(250ms)).Where(xs => xs.Length >= 2)`를 한국어 한 문장으로 읽으시오.
3. "HP > 0이고 탄약 > 0일 때만 발사 버튼 활성화"를 어떤 연산자로 만드는지 코드 골격을 쓰시오.
4. UniRx 코드의 `Buffer`·`Throttle`을 R3로 옮길 때 각각 무엇으로 바꾸는가?

> ✅ **해설**
>
> 1. 입력 `--a-b-c--------`에 대해 Debounce는 잠잠해진 뒤 `c` 한 번, ThrottleFirst는 시작 즉시 `a` 한 번.
> 2. "클릭이 250ms 잠잠해질 때까지 모아서, 2개 이상이면 더블클릭."
> 3. `hp.CombineLatest(ammo, (h, a) => h > 0 && a > 0).DistinctUntilChanged().Subscribe(can => button.interactable = can)`
> 4. `Buffer→Chunk`(결과 `.Count`→`.Length`), `Throttle→Debounce`(R3에 Throttle 없음).

- **메서드를 채웠는데도 무반응**: 가장 먼저 의심할 곳 — 해당 Setup이 아직 빈 채이거나, 코드를 다른 메서드에 넣었거나, `.AddTo(this)`/`.Subscribe(...)`를 빠뜨린 경우. `Start()`가 세 Setup을 호출하는 줄은 스켈레톤에 이미 있음.
- **흔한 오류**: Inspector 연결 누락(버튼·텍스트 무반응 — 가장 흔함, 스켈레톤 씬엔 연결돼 있으나 학생이 오브젝트를 새로 만들었다면 직접 연결), 한글 텍스트 깨짐(`R3Lecture-Student-1-Basics` 패키지를 먼저 임포트했는지 확인 — 폰트 포함), EventSystem 부재(클릭 무시).
- 강사 씬이 꼬이면 `Tools > Reactive Study > Create All Scenes`로 재생성(씬은 git 비커밋). 학생 코드가 꼬이면 `R3Lecture-Student-1-Basics` 패키지 재임포트로 스켈레톤 초기화(작성 중 코드는 백업 안내).
- 더블클릭이 잘 안 잡히면 "더 빠르게 두 번" — 250ms 안에 두 클릭이 들어와야 한다. 천천히 누르면 의도대로 무반응.
- 막힌 학생에게는 모범 코드(`OperatorsDemo.cs`)의 해당 부분을 화면 공유로 보여 주되 **복붙으로 건너뛰게 하지는 말 것**(타이핑이 학습). 에러도 교보재 — 그 자리에서 함께 디버깅.
