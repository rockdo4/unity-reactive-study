# 02. R3 시작하기 — 강사용

> 학생 교안: [02. R3 시작하기 — Observable 만들기](../교안/02.%20R3%20시작하기%20—%20Observable%20만들기.md)
> 데모: 기초 데모 씬 패널 1 · `Assets/Scripts/ReactiveBasics/CreateStreamsDemo.cs`

## 1. 강사 사전 학습 · 리마인드

- **설치 2단계의 이유를 자기 말로** — R3 본체는 범용 .NET 라이브러리(NuGet 배포), R3.Unity는 유니티 통합(UPM git URL). "유니티에서 NuGet을 쓰는 표준 통로가 NuGetForUnity"까지가 세트. 이 저장소는 설치가 끝나 있으므로 수업에서는 **절차를 보여 주기만** 하고 실제 설치는 안 함(시간 절약). 직접 시연하고 싶다면 별도 빈 프로젝트를 미리 준비.
- **"Subscribe는 등록"** — 1차시의 "`+=`는 명단 등록" 비유를 그대로 재사용. 학생이 가장 많이 하는 오해가 "Subscribe하면 실행되는 것"임. `Range`는 구독 즉시 흐르니 오해를 강화할 수 있음 — "Range는 구독 순간 다 흘려보내고 끝나는 특수한(콜드) 경우"라고 명시.
- **3채널(OnNext/OnErrorResume/OnCompleted)** — 이 차시에서는 "채널이 **있다**"까지만. `OnErrorResume`의 진짜 의미(구독 유지)는 5차시에서 다룸. 여기서 깊이 들어가면 시간이 샘.
- **콜드/핫** — 정의 논쟁으로 들어가지 말 것. "팩토리는 구독마다 새로(콜드), Subject·클릭은 흐르는 데 합류(핫)" 두 문장이면 충분. `Share()`는 3차시로 미룸.
- **데모 코드 예열** — `CreateStreamsDemo.cs`의 Interval 토글(수동 `Dispose`)이 4차시 복선임. "Dispose가 곧 해제"를 여기서 처음 보여 주고 4차시에서 회수하는 구조.

## 2. 수업 진행

### 도입 (~3분)

> 💡 "지난 시간에 '구독'이라는 말을 정의 없이 썼죠. 오늘은 진짜로 구독해 봅니다 — 그리고 스트림을 **만드는** 다섯 가지 방법을 배웁니다."

### 1장. 설치 (~7분)

- 표(R3 본체=NuGet / R3.Unity=git URL)를 띄우고 절차만 읽음. **이 저장소는 설치 완료**임을 알리고, `Assets/Packages/`(NuGet DLL)와 Package Manager의 R3.Unity를 에디터에서 직접 보여 줄 것.
- **⚠️ 함정**: "버전 어떻게 맞춰요?" — git URL 태그와 코어 NuGet 버전을 동일 버전으로 맞춘다고 답. 최신 버전 재확인은 운영 가이드 가드레일대로.

### 2장. 첫 구독 (~8분)

- `Range(1,3).Subscribe(...)` 라이브 코딩 → 콘솔 확인.
- `Subscribe` 반환값을 `IDisposable` 변수로 받아 보이기 — "이게 구독 그 자체. 4차시의 주인공."
- 풀 버전 Subscribe(3채널)를 한 번 보여 주되 "지금은 '있다'만".

### 3장. 팩토리 (~7분)

- `Range`/`Timer`/`Interval`/`Return`을 마블과 함께. `Interval`은 **멈추는 법(Dispose)** 까지가 한 세트.
- **칠판**: `Range ──1─2─3─4─| · Timer ────●| · Interval ──●──●──●──▶`

### 4장. Subject (~5분)

- `event` 코드와 나란히 띄우고 1:1 대응(선언↔필드, Invoke↔OnNext, +=↔Subscribe).
- 캡슐화 관례(밖에는 `Observable<T>`로) — "event가 밖에서 Invoke 안 되는 것과 같은 이유".

### 5~6장. ReactiveProperty · FromEvent (~5분)

- RP 차이 3가지(현재값 보유·구독 즉시 통지·중복 억제)를 Subject와 대비.
- FromEvent는 "기존 코드와의 다리, `-=` 자동"이 전부. 시그니처 문법(`h => e += h`)은 어렵게 느끼므로 "외워 쓰는 관용구"라고 안심시킬 것.

### 7장. 따라하기 — 데모 패널 1 (~20분)

1. 씬 생성기 실행(처음이면 메뉴 위치부터) → Play.
2. 버튼 5개를 순서대로: Range(완료 로그 주목) → Timer → Interval 시작/중지(**중지가 Dispose임을 코드로 확인**) → Subject → FromEvent.
3. `CreateStreamsDemo.cs`를 프로젝터에 띄워 위에서부터 같이 읽기 — "각 버튼이 교안 몇 절일까요?"를 학생에게 묻는 방식.
4. 시간이 남으면: `Range(1, 4)`를 `Range(1, 10)`으로 고쳐 핫리로드 — 작은 수정을 직접 해 보는 경험.

- **⚠️ 함정**: Interval을 여러 번 시작하는 학생이 나옴(토글이라 안 생기지만 코드를 고치다 보면) — "스트림은 만들 때마다 새것"이라는 콜드 개념으로 설명 기회로 삼을 것.

### 정리 (~5분)

- 만들기 5종 복기 + 체크리스트.

## 3. 예상 Q&A

> ❓ **NuGet 없이 DLL만 복사하면 안 되나요?**
> 기술적으로는 되지만 의존 패키지(Microsoft.Bcl.TimeProvider 등) 누락·버전 충돌을 손으로 관리하게 됨. NuGetForUnity가 그걸 자동화하는 표준 도구.

> ❓ **Subject랑 ReactiveProperty 중 뭘 써야 하나요?**
> "현재값"이 의미 있으면 RP(체력·골드·설정값), 순간의 사건이면 Subject(공격 명중·아이템 드랍). "지금 값이 뭐야?"라는 질문이 성립하면 RP.

> ❓ **Observable이 인터페이스가 아니라 클래스인 이유는?**
> R3가 모든 구독을 베이스 클래스에서 추적·관리하기 위해서(구독 누수 진단에 유리). 깊이는 들어가지 말고 "그래서 Tracker 같은 도구가 가능하다" 정도로.

## 4. 이해도 퀴즈

1. R3 설치가 두 단계(NuGet + git URL)인 이유를 쓰시오.
2. `Subscribe`가 "실행"이 아니라 "등록"인 이유를 `+=` 비유로 설명하시오.
3. `Interval`로 만든 무한 스트림을 멈추는 코드를 쓰시오.
4. `Subject<int>`와 `ReactiveProperty<int>`의 차이를 두 가지 이상 쓰시오.

> ✅ **해설**
>
> 1. R3 본체는 범용 .NET 라이브러리라 NuGet으로 배포되고, 유니티 전용 통합(AddTo·프로바이더 등)은 별도 패키지 R3.Unity로 UPM(git URL) 배포되기 때문.
> 2. 명단에 이름을 올리는 것과 같음 — 콜백은 값이 흐를 때 실행됨. (`Range`처럼 구독 즉시 흐르는 콜드 스트림은 등록과 동시에 값이 오는 특수 경우.)
> 3. `IDisposable sub = Observable.Interval(...).Subscribe(...); sub.Dispose();`
> 4. RP는 현재값을 보유(`.Value`), 구독 즉시 현재값 1회 통지, 같은 값 중복 억제. Subject는 셋 다 없음.
