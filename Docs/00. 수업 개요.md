# 유니티 리액티브 프로그래밍 (R3) 수업 개요

리액티브 프로그래밍의 **일반 이론**과 **R3 사용법**을 다루는 독립 강의입니다. 디자인 패턴 강의의 부속이 아니라, 옵저버 패턴 기초부터 R3 실전 활용까지 자족적으로 다룹니다. 수업은 **이론(교안) → 데모** 순서로 진행합니다.

> 기준 버전: **R3** (`com.cysharp.r3`, 수업 시점 최신 버전) · Unity 6.3 / URP 17
> 공식 저장소: <https://github.com/Cysharp/R3>

## 이 자료의 구성

| 구분        | 폴더      | 용도                                             |
| ----------- | --------- | ------------------------------------------------ |
| 강의 교안   | `교안/`   | 학생에게 노션으로 배포하는 순수 이론(01~05)      |
| 강사용 자료 | `강사용/` | 수업 준비·진행용 (사전 학습, 진행 스크립트, Q&A) |
| 데모 가이드 | `데모/`   | 데모 씬 구성·시연 대본·씬 자동 생성 절차         |

- 학생 **교안(01~05)**은 개념·이론·일반 코드 예제만 담은 **순수 이론**이며, 각 장은 독립적으로 읽힙니다(데모 따라하기·씬·저장소 경로 인용 없음).
- 데모 시연 대본·따라하기는 **강사용 문서**에 있습니다. 차시별로 이론 문서와 따라하기 문서로 나뉩니다(01은 이론만, 06은 따라하기만).

> ℹ️ **강사가 R3를 처음 접한다면** — [강사용/00. 강사 사전 학습 가이드](강사용/00.%20강사%20사전%20학습%20가이드.md)의 「강사 사전 학습 로드맵」부터 읽으세요. 수업 전 자기 학습 순서와 자료가 정리되어 있습니다.

## 커리큘럼

데모는 **두 씬·두 패키지**로 배포합니다 — 1부(R3 기초)는 5개 섹션을 한 씬에 담은 **통합 데모 씬 하나**, 2부는 클리커 게임 씬 하나입니다. 학생은 빈 URP 프로젝트에 R3·UniTask·Input System 을 설치한 뒤, **1부 기초 패키지를 수업 시작 때 가장 먼저** 임포트하고(공통 폰트 포함) 2부에 들어갈 때 클리커 패키지를 임포트합니다(→ [데모 프로젝트 준비](#데모-프로젝트-준비)).

### 1부 — 기본 개념 및 사용법

1부 전체는 **통합 데모 씬 하나**(`ReactiveBasics`, 5섹션 2열)로 시연하고, **패키지 하나**(`R3Lecture-Student-1-Basics`, 공통 폰트 포함)로 배포합니다. 이론(교안)을 마친 뒤 통합 씬의 해당 섹션 데모를 보는 식으로 차시를 오갑니다.

| 차시 | 교안                                                                                         | 통합 씬 섹션          | 한 줄 요약                                         |
| ---- | -------------------------------------------------------------------------------------------- | --------------------- | -------------------------------------------------- |
| 01   | [리액티브 프로그래밍이란](교안/01.%20리액티브%20프로그래밍이란.md)                           | – (이론)              | 옵저버 패턴의 일반화, 스트림, 마블 다이어그램      |
| 02   | [R3 시작하기 - Observable 만들기](교안/02.%20R3%20시작하기%20-%20Observable%20만들기.md)     | §1 생성               | 설치, Subscribe, 팩토리·Subject·ReactiveProperty   |
| 03   | [연산자 - 스트림을 다듬고 합성하기](교안/03.%20연산자%20-%20스트림을%20다듬고%20합성하기.md) | §2 연산자             | Where/Select/Debounce/Chunk/Merge/CombineLatest 등 |

### 2부 — R3로 미니 게임 만들기 (2차시 권장)

2부는 클리커 게임을 라이브 코딩으로 완성합니다. 데모 씬 `ReactiveClicker`(스켈레톤) + 패키지 `R3Lecture-Student-2-Clicker`로 배포합니다. 학생 교안은 없고(순수 실습), 진행 대본은 [06. 클리커 따라하기 - 강사용](강사용/06.%20클리커%20따라하기%20-%20강사용.md)에 있습니다.

| 차시  | 데모 씬           | 학생 패키지                   | 한 줄 요약                               |
| ----- | ----------------- | ----------------------------- | ---------------------------------------- |
| 06 ①② | `ReactiveClicker` | `R3Lecture-Student-2-Clicker` | 1부에서 배운 것만으로 클리커 게임을 완성 |

> 1부 통합 씬(`ReactiveBasics`, 5개 섹션 2열)과 2부 클리커 모두 핵심 R3 코드를 비운 **스켈레톤**(라이브 코딩용)입니다 — 수업 중 강사와 한 줄씩 함께 작성합니다.

## 수업 운영 방식

- 수업은 **이론(교안) → 데모** 순서를 따릅니다. 어휘 없이 코드부터 가지 않습니다.
- 1부 데모는 **통합 씬 하나**입니다 — 그날 이론에 해당하는 섹션을 통합 씬에서 함께 작성·확인합니다(섹션 §1~§5).
- 1·2부 모두 학생이 강사를 따라 **직접 코드를 작성**하는 라이브 코딩 수업입니다. 막혔을 때는 강사 프로젝트의 완성본(1부 `Assets/Scripts/ReactiveBasics/`, 2부 `Assets/Scripts/ClickerGame/`)을 대조용으로 씁니다.

## 데모 프로젝트 준비

배포 모델이 둘로 나뉩니다 — **학생은 빈 프로젝트에 패키지를 임포트**하고, **강사는 이 저장소**로 씬·패키지를 생성합니다.

### 학생 — 빈 프로젝트 + 패키지 임포트

1. **빈 URP 프로젝트**를 Unity 6.3으로 만들고, R3(NuGetForUnity + `R3.Unity` git URL)·UniTask(git URL)·Input System 을 설치합니다.
    - 설치 절차(NuGetForUnity + git URL)는 [교안 02](교안/02.%20R3%20시작하기%20-%20Observable%20만들기.md)의 1장에서 단계별로 다룹니다.
2. **1부 기초 패키지 `R3Lecture-Student-1-Basics` 를 수업 시작 때 가장 먼저** 임포트합니다(Pretendard 한글 폰트 + TMP 설정 포함 — 한글 깨짐 방지).
3. 2부에 들어갈 때 **클리커 패키지 `R3Lecture-Student-2-Clicker`** 를 임포트합니다.

> 학생은 저장소를 클론하지 않습니다 — 교안은 노션 공유 페이지로, 데모는 `.unitypackage` 2종으로 받습니다.

### 강사 — 이 저장소

1. 이 저장소를 클론하고 Unity 6.3으로 엽니다. R3는 **이미 설치되어 있습니다**(NuGet `Assets/Packages/` + Package Manager `R3.Unity`).
2. 강사용 데모 씬을 에디터 메뉴로 생성합니다 — `Tools > Reactive Study > Create All Scenes`(통합 기초 + 클리커 두 씬). 생성기가 정본이며, 강사용 씬은 git에 커밋하지 않습니다.
3. 학생 배포물은 `Tools > Reactive Study > Student Package > Create All Student Scenes` 로 학생용 씬 2개를 만든 뒤 `Export All Student Packages` 로 `.unitypackage` 2종을 내보냅니다.
4. 데모 코드 위치: `Assets/Scripts/ReactiveBasics/`(1부), `Assets/Scripts/ClickerGame/`(2부).

자세한 내용은 도구 가이드를 참조하세요 — [씬 생성·패키지 도구 가이드](데모/씬%20생성·패키지%20도구%20가이드.md)(씬에 무엇이 들었나·씬 생성 절차·체크리스트·문제 해결).
