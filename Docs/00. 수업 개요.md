# 유니티 리액티브 프로그래밍 (R3) 수업 개요

리액티브 프로그래밍의 **일반 이론**과 **R3 사용법**을 다루는 독립 강의입니다. 디자인 패턴 강의의 부속이 아니라, 옵저버 패턴 기초부터 R3 실전 활용까지 자족적으로 다룹니다. 모든 차시는 **이론 → 따라하기** 순서로 진행합니다.

> 기준 버전: **R3** (`com.cysharp.r3`, 수업 시점 최신 버전) · Unity 6.3 / URP 17
> 공식 저장소: <https://github.com/Cysharp/R3>

## 이 자료의 구성

| 구분        | 폴더      | 용도                                             |
| ----------- | --------- | ------------------------------------------------ |
| 강의 교안   | `교안/`   | 수업 중 학생과 함께 보는 자료                    |
| 강사용 자료 | `강사용/` | 수업 준비·진행용 (사전 학습, 진행 스크립트, Q&A) |
| 데모 가이드 | `데모/`   | 데모 씬 구성·시연 대본·씬 자동 생성 절차         |

각 차시는 **이론 문서**(개념·어휘)와 **따라하기 문서**(데모 시연·코드)로 나뉩니다(01·06은 한 종류뿐). 강사용 문서가 둘을 짝지어 안내합니다.

> ℹ️ **강사가 R3를 처음 접한다면** — [강사용/00. 강사 사전 학습 가이드](강사용/00.%20강사%20사전%20학습%20가이드.md)의 「강사 사전 학습 로드맵」부터 읽으세요. 수업 전 자기 학습 순서와 자료가 정리되어 있습니다.

## 커리큘럼

각 차시는 **독립 데모 씬**과 (해당 시) **학생 배포용 패키지**(`.unitypackage`)를 가집니다. 학생은 빈 URP 프로젝트에 R3·Input System 을 설치한 뒤, **공통 패키지(00-Common)를 가장 먼저** 임포트하고 차시별 패키지를 진도에 맞춰 임포트합니다(→ [데모 프로젝트 준비](#데모-프로젝트-준비)).

## 수업 운영 방식

- 각 차시는 **이론 설명 → 강사 시연·학생 따라하기** 순서를 따릅니다. 어휘 없이 코드부터 가지 않습니다.
- 1부 데모는 차시마다 **독립 씬**입니다(예: 03차시는 `03_Operators`만 열면 됩니다). 그날 배운 씬만 보면 됩니다.
- 2부는 학생이 교안의 단계를 따라 **직접 코드를 작성**하는 따라하기 수업입니다. 완성본 씬(`06_ReactiveClicker`)은 막혔을 때 비교·확인용입니다.

## 데모 프로젝트 준비

배포 모델이 둘로 나뉩니다 — **학생은 빈 프로젝트에 패키지를 임포트**하고, **강사는 이 저장소**로 씬·패키지를 생성합니다.

### 학생 — 빈 프로젝트 + 패키지 임포트

1. **빈 URP 프로젝트**를 Unity 6.3으로 만들고, R3(NuGetForUnity + `R3.Unity` git URL)와 Input System 을 설치합니다.
    - 설치 절차(NuGetForUnity + git URL)는 [교안 02](교안/02.%20R3%20시작하기%20—%20Observable%20만들기.md)의 1장에서 단계별로 다룹니다.
2. **공통 패키지 `R3Lecture-Student-00-Common` 을 가장 먼저** 임포트합니다(Pretendard 한글 폰트 + TMP 설정 — 한글 깨짐 방지).
3. 진도에 맞춰 **차시별 패키지**(`R3Lecture-Student-02-CreateStreams` …)를 하나씩 임포트하고, 임포트된 씬을 엽니다.

> 학생은 저장소를 클론하지 않습니다 — 교안은 노션 공유 페이지로, 데모는 차시별 `.unitypackage` 로 받습니다.

### 강사 — 이 저장소

1. 이 저장소를 클론하고 Unity 6.3으로 엽니다. R3는 **이미 설치되어 있습니다**(NuGet `Assets/Packages/` + Package Manager `R3.Unity`).
2. 강사용 데모 씬을 에디터 메뉴로 생성합니다 — `Tools > Reactive Study > Create All Scenes`(또는 차시별 메뉴). 생성기가 정본이며, 강사용 씬은 git에 커밋하지 않습니다.
3. 학생 배포물은 `Tools > Reactive Study > Student Package > Create All Student Scenes` 로 학생용 씬을 만든 뒤 `Export All Student Packages` 로 `.unitypackage` 를 내보냅니다.
4. 데모 코드 위치: `Assets/Scripts/ReactiveBasics/`(1부), `Assets/Scripts/ClickerGame/`(2부).

자세한 내용은 도구 가이드를 참조하세요 — [씬 생성·패키지 도구 가이드](데모/씬%20생성·패키지%20도구%20가이드.md)(씬에 무엇이 들었나·씬 생성 절차·체크리스트·문제 해결).
