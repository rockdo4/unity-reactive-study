# 타이밍 바 — 빈 씬에서 만드는 R3 미니게임 (과제)

> 이 과제는 강의에서 배운 R3를 **스스로 조립해** 작은 게임 하나를 완성하는 것입니다. 강사가 채워 주는 빈칸(스켈레톤)은 없습니다 — **빈 씬에서 코드와 화면을 직접** 만듭니다. 먼저 배포된 **완성본(윈도우 빌드)** 을 한 번 실행해 "이렇게 동작하는 걸 만든다"를 확인하고 시작하세요.

## 만들 게임

좌우로 끝없이 왕복하는 **마커**가 트랙 가운데 **존**을 지날 때 버튼(또는 Space)을 눌러 점수를 얻는 한 줄 타이밍 게임입니다.

- 트랙 가운데에 **Good 존**(넓음)과 그 안에 겹친 **Perfect 존**(좁음)이 있습니다.
- START → **3·2·1 카운트다운** 후 라운드가 시작됩니다.
- 마커가 존에 있을 때 누르면: **Perfect = +3, Good = +1, 존 밖 = Miss(콤보 0)**. 콤보가 길수록 점수 배수가 올라갑니다.
- 한 라운드는 **정해진 횟수(예: 12회)** 를 치면 끝나고, **최종 점수·최고 콤보** 가 표시됩니다. 다시 시작할 수 있습니다.
- 마커가 존에 **들어오는 순간** 트랙 색이 바뀌어 타이밍을 시각적으로 돕습니다.

## 학습 목표

이 한 게임으로 강의의 R3 도구를 종합합니다.

- **프레임 스트림** `Observable.EveryUpdate()` 으로 마커를 움직인다(게임 루프를 `Update()` 없이).
- **입력 시점 샘플링** `WithLatestFrom` 으로 "누른 순간"의 마커 위치를 가져온다.
- **상태머신** `Select` + `DistinctUntilChanged` 으로 존 진입 순간만 감지한다.
- **시퀀스 종료** `Take(n)` 으로 라운드를 정해진 횟수에 끝낸다.
- **비동기 흐름** `SubscribeAwait(AwaitOperation.Drop)` + `UniTask.Delay` 로 카운트다운 → 플레이 → 결과를 잇는다.
- **수명 관리** `AddTo` · `CompositeDisposable` 로 누수 없이 정리하고 `ObservableTracker` 로 검증한다.

> 핵심 규칙: **`Update()`·코루틴·상태 플래그를 쓰지 않습니다.** 모든 흐름을 스트림으로 표현하세요.

## 준비물

- 수업에서 셋업한 프로젝트(빈 URP + **R3 · UniTask · Input System** 설치, 한글 폰트 포함)면 충분합니다.
- 새 씬 하나를 만들고 거기서 시작합니다.

## 1단계 — 화면 준비 (가볍게)

R3 구현에 집중할 수 있게 화면은 최소로 만듭니다. `Canvas`(Screen Space - Overlay) 하나에 아래 요소를 배치하세요. EventSystem은 **Input System UI Input Module** 을 씁니다(이 프로젝트는 New Input System).

| 요소                                                             | 종류                           | 비고                                          |
| ---------------------------------------------------------------- | ------------------------------ | --------------------------------------------- |
| 트랙                                                             | Image (가로 막대)              | 화면 중앙. 마커가 이 위를 움직인다            |
| Good 존 / Perfect 존                                             | Image (트랙의 자식, 중앙)      | 고정 기준선. Perfect가 더 좁고 Good 안에 겹침 |
| 마커                                                             | Image (트랙의 자식, 중앙 앵커) | 좌우로 움직일 핸들                            |
| HIT 버튼                                                         | Button                         | Space로도 칠 수 있게 할 것                    |
| START 버튼                                                       | Button                         | 라운드 시작                                   |
| 점수 / 콤보 / 진행도(n/12) / 판정(Perfect·Good·Miss) / 상태·결과 | Text(TMP)                      | 표시용                                        |

> 💡 마커와 존을 **중앙 앵커(0.5, 0.5)** 로 두면 좌우 대칭으로 움직이고 겹치기 쉽습니다. 마커 위치는 위상값 `0~1`(0=왼쪽 끝, 0.5=중앙, 1=오른쪽 끝)을 트랙 폭에 맞춘 `anchoredPosition.x` 로 변환합니다.

**Model / Presenter 나누기**: 게임 규칙(판정·점수·콤보)은 순수 C# 클래스 `Model`에, 입력·프레임·화면 갱신은 `MonoBehaviour` `Presenter`에 둡니다. Model이 UI를 모르면 규칙을 단위 테스트할 수 있고, 화면을 바꿔도 규칙이 안 깨집니다.

## 2단계 — 마커를 움직인다 (프레임 스트림)

`Observable.EveryUpdate()` 를 구독해 매 프레임 마커 위상을 계산하고(`Mathf.PingPong` 이 0→1→0 왕복에 편합니다) 위치를 갱신합니다.

- 위상값을 **`ReactiveProperty<float>`** 에 담아 두세요. 잠시 뒤 입력 판정이 이 "현재 위상"을 샘플링합니다.
- 전역·무한 스트림이므로 **반드시 `.AddTo(this)`**.

✅ 확인: Play하면 마커가 좌우로 왕복한다.

## 3단계 — 존에 들어오는 순간 강조 (상태머신)

위상 → 존(None/Good/Perfect) 으로 바꾸고, **바뀌는 순간에만** 트랙 색을 바꿉니다.

- `위상스트림.Select(위상 → 존).DistinctUntilChanged().Subscribe(존 → 트랙색 변경)`

✅ 확인: 마커가 가운데를 지날 때만 트랙 색이 바뀐다(매 프레임이 아니라 진입 순간 한 번).

## 4단계 — 누른 순간을 판정한다 (입력 합성 + 샘플링)

- HIT 버튼 `OnClickAsObservable()` 과 Space 키(`EveryUpdate` 안에서 `Keyboard.current.spaceKey.wasPressedThisFrame`)를 **`Merge`** 로 하나의 입력 스트림으로 만듭니다.
- 입력 스트림에 **`WithLatestFrom(위상)`** 을 걸어 "누른 순간의 마커 위치"를 가져와 Model에 넘겨 판정합니다.
- 판정 결과로 점수·콤보를 갱신하고(콤보는 연속 성공 시 증가, Miss면 0), 화면에 Perfect/Good/Miss를 표시합니다.

> 💡 콤보 누적은 Model 안에서 직접 다뤄도 되고, 판정 스트림을 **`Scan`** 으로 접어 만들어도 됩니다. 둘 중 하나를 고르세요.

✅ 확인: 존에서 누르면 점수가 오르고 콤보가 쌓이며, 존 밖에서 누르면 Miss로 콤보가 0이 된다.

## 5단계 — 라운드 흐름 (비동기 + 시퀀스 종료)

START를 누르면 한 판이 흐릅니다: **카운트다운 → 플레이(N회) → 결과**.

- START `OnClickAsObservable().SubscribeAwait(async (_, ct) => { ... }, AwaitOperation.Drop)` — 진행 중 START 재클릭은 `Drop`으로 무시됩니다.
- 카운트다운은 `UniTask.Delay` 로 3·2·1을 띄웁니다.
- 라운드 입력 스트림에 **`Take(N)`** 을 걸면 N회 시도 후 자동으로 끝납니다(완료). 그 완료를 `await`(예: `WaitAsync`)로 기다렸다가 결과를 표시합니다.
- 라운드 동안만 사는 구독은 **라운드용 `CompositeDisposable`** 에 담아 라운드가 끝나면 정리하세요(영속 구독과 구분).

✅ 확인: START → 카운트다운 → 12회를 치면 결과(점수·최고 콤보)가 뜨고 다시 시작할 수 있다.

## 6단계 — 누수 0 검증 (필수)

- 전역 스트림(`EveryUpdate` 등)과 라운드 구독이 모두 정리되는지 `Window > Observable Tracker` 로 확인합니다.
- Tracker를 켜고 한 판 플레이한 뒤 Presenter를 삭제(또는 씬을 비움) → **남은 구독이 0이면 합격**. 남아 있으면 `AddTo`/`CompositeDisposable` 가 빠진 곳입니다.

## R3 연산자 힌트 (단계 ↔ 도구)

| 단계           | 도구                                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------------------- |
| 2. 마커 이동   | `Observable.EveryUpdate`, `ReactiveProperty<float>`, `AddTo`                                         |
| 3. 존 강조     | `Select`, `DistinctUntilChanged`                                                                     |
| 4. 입력 판정   | `OnClickAsObservable`, `Merge`, `WithLatestFrom`, (`Scan`)                                           |
| 5. 라운드 흐름 | `SubscribeAwait`(`AwaitOperation.Drop`), `UniTask.Delay`, `Take`, `WaitAsync`, `CompositeDisposable` |
| 6. 수명        | `AddTo`, `CompositeDisposable`, `ObservableTracker`                                                  |
| 표시           | `ReactiveProperty` + `Subscribe`(값→텍스트)                                                          |

## 완료 기준

- [ ] 윈도우 빌드와 같은 흐름으로 동작한다(카운트다운 → 마커 왕복 → 존 판정/콤보/점수 → N회 후 결과 → 다시 시작).
- [ ] `Update()` · 코루틴 · 상태 플래그를 쓰지 않았다(모든 흐름이 스트림).
- [ ] 게임 규칙(판정·점수·콤보)이 순수 C# Model에 있고, 입력·표시는 Presenter에 있다.
- [ ] `Observable Tracker` 에서 씬을 나가면 남은 구독이 0이다.
- [ ] Console에 에러·경고가 없다.

## 제출물

- 작성한 스크립트(Model·Presenter).
- 플레이 영상(한 판 — 카운트다운부터 결과까지).
- Observable Tracker 스크린샷(씬을 나간 뒤 남은 구독 0).

## 도전 과제 (선택)

- **제한시간 모드**: N회 종료 대신(또는 함께) 제한시간을 두고 `TakeUntil(Observable.Timer(...))` 로 라운드를 끝낸다.
- **난도 상승**: 라운드(또는 시간)가 지날수록 마커 속도를 단계적으로 올린다.
- **시간 의존 규칙 테스트**: 가상 시간(TimeProvider)으로 라운드 흐름을 결정론적으로 검증한다.
