# 데모 구성 가이드

이 강의의 데모 씬 두 개(`ReactiveBasics`·`ReactiveClicker`)에 **무엇이 들어 있고 어떻게 동작하는지**를 정리한다. 씬은 에디터 메뉴로 자동 생성하며(→ [자동 생성 가이드](자동%20생성%20가이드.md)), 수업 중 조작 대본은 [시연 진행 가이드](시연%20진행%20가이드.md)에 있다.

> 연결 문서: [00. 수업 개요](../00.%20수업%20개요.md) · [강사용 운영 가이드](../강사용/00.%20강의%20운영%20가이드.md) · 자매 문서 [시연 진행 가이드](시연%20진행%20가이드.md) · [자동 생성 가이드](자동%20생성%20가이드.md)

## 개요

| 씬                | 경로                                  | 대응 교안 | 역할                                    |
| ----------------- | ------------------------------------- | --------- | --------------------------------------- |
| `ReactiveBasics`  | `Assets/Scenes/ReactiveBasics.unity`  | 02~05     | 1부 개념·사용법 데모(패널 5개)          |
| `ReactiveClicker` | `Assets/Scenes/ReactiveClicker.unity` | 06        | 2부 따라하기 게임 **완성본**(시연·확인) |

- 씬 파일은 git에 커밋하지 않는다 — **에디터 메뉴로 매번 생성**한다(생성기가 정본). 코드가 바뀌면 씬도 다시 생성한다.
- 모든 텍스트는 한글 폰트(`Pretendard`)를 코드에서 자동 주입한다(아래 [한글 폰트](#한글-폰트)).

## 씬 1 — 리액티브 기초 데모 (`ReactiveBasics`)

생성기 `Assets/Scripts/ReactiveBasics/Editor/ReactiveBasicsSceneCreator.cs`. 한 Canvas 위에 패널 5개를 배치하고, 각 패널마다 데모 컴포넌트 1개와 UI 참조를 코드로 연결한다(인스펙터 수동 연결 불필요).

| 패널                 | GameObject         | 컴포넌트(`Assets/Scripts/ReactiveBasics/`) | 교안 | 핵심 UI                                                                                        |
| -------------------- | ------------------ | ------------------------------------------ | ---- | ---------------------------------------------------------------------------------------------- |
| 1. Observable 만들기 | `Panel1_Create`    | `CreateStreamsDemo.cs`                     | 02   | 버튼 `Range(1,4)`·`Timer 2초`·`Interval 시작`·`Subject.OnNext`·`event 발생 (FromEvent)` + 로그 |
| 2. 연산자            | `Panel2_Operators` | `OperatorsDemo.cs`                         | 03   | 더블클릭 안내 · 검색 입력창 · `탭` 버튼 · 쿨다운 안내                                          |
| 3. 구독 수명         | `Panel3_Lifetime`  | `LifetimeDemo.cs` (+ `LifetimeProbe`)      | 04   | 버튼 `구독 추가`·`모두 해제`·`누수 만들기`·`활성 구독 조회`·`프로브 파괴`                      |
| 4. 바인딩            | `Panel4_Binding`   | `BindingDemo.cs`                           | 05   | `피해/회복`·`MP -5/+5`·`스킬 사용` + HP/MP/상태 표시                                           |
| 5. 비동기            | `Panel5_Async`     | `AsyncDemo.cs`                             | 05   | `순차 로드 (Sequential)`·`드롭 로드 (Drop)` + 로그(1건당 1.5초)                                |

### 패널별 스트림·연산자·수명

각 `Subscribe`는 달리 적지 않으면 `.AddTo(this)`로 컴포넌트 수명에 묶인다(오브젝트 파괴 시 자동 해제).

| 패널 | 스트림·연산자 사슬(요지)                                                                                                                                                                          | 수명(AddTo 대상)                                  |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| 1    | `Range`/`Timer`/`Interval` 팩토리, `Subject.OnNext`, `FromEvent`(구독 해제 시 `-=` 자동)                                                                                                          | `this`                                            |
| 2    | 더블클릭 `EveryUpdate→Where→Share` + `Chunk(Debounce 250ms)→Where(≥2)`; 검색 `Debounce 500ms→DistinctUntilChanged`; 3연타 `Chunk(500ms,3)→Where(≥3)`; 쿨다운 `ThrottleFirst(2s)`                  | `this`                                            |
| 3    | 관리 구독 `Interval(5s)→AddTo(CompositeDisposable)`; **누수** `Interval(5s)` (AddTo 없음 — 영구 실행); `ObservableTracker`로 활성 구독 조회; 프로브 `Interval(3s)→AddTo(this)`(파괴 시 자동 해제) | `m_Disposables`(관리), 없음(누수), `this`(프로브) |
| 4    | `SerializableReactiveProperty<int>` HP(100)·MP(50); `Select→DistinctUntilChanged`로 상태 라벨; `CombineLatest`로 스킬 버튼 `interactable`                                                         | `this`                                            |
| 5    | `SubscribeAwait(..., AwaitOperation.Sequential)` / `... Drop` (로드 1건당 1.5초, `ct`는 구독 해제 시 자동 취소)                                                                                   | `this`                                            |

> 패널 3은 의도적으로 **누수 만들기** 버튼을 둔다 — `AddTo`/`Dispose` 없는 전역 `Interval` 구독은 오브젝트와 무관하게 영원히 산다. `Window > Observable Tracker`와 `활성 구독 조회`로 눈에 보이는 게 4차시의 핵심 시연이다(→ [시연 진행 가이드](시연%20진행%20가이드.md)).

## 씬 2 — 리액티브 클리커 (`ReactiveClicker`)

생성기 `Assets/Scripts/ClickerGame/Editor/ClickerGameSceneCreator.cs`. 교안 06의 **완성본**을 한 번에 만든다(수업에서는 학생이 단계별로 직접 만들고, 이 씬은 확인·시연용 보조 자료).

| GameObject             | 컴포넌트 / 역할                                                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `ClickerPresenter`     | `ClickerPresenter.cs` — 입력→Model 호출, Model의 `ReactiveProperty`→UI 바인딩                                                  |
| (Model, 비-GameObject) | `ClickerModel.cs` — 순수 C# 상태·규칙(`Gold`·`GoldPerClick`·`IsFever`...). Presenter가 `new`로 생성, `OnDestroy`에서 `Dispose` |
| `ClickButton` "클릭!"  | 클릭 스트림 — 본체·콤보·보너스 3곳에서 `Share`로 공유                                                                          |
| `FeverButton` "피버!"  | `ThrottleFirst(15초)` 쿨다운 게이트 → `TryStartFever`                                                                          |
| `ComboText`            | `Select(1)`+`Merge(Debounce(0.8s)→0)`+`Scan` 누적 콤보                                                                         |
| `StatusPanel`          | `GoldText`·`StatsText`·`ClickUpgradeButton`("클릭 강화/비용 10")·`AutoUpgradeButton`("도우미 고용/비용 25")·`EventLogText`     |
| `FeverStateText`       | `IsFever` 바인딩(피버 중/쿨다운 안내 + 클릭 버튼 색)                                                                           |

> 구매 가능 여부(`CanBuyClickUpgrade`·`CanBuyAutoUpgrade`)는 Model에서 `Gold.CombineLatest(비용)`으로 파생되어 버튼 `interactable`에 그대로 바인딩된다 — Presenter에 분기 로직이 없다. Model 규칙의 EditMode 단위 테스트는 `Assets/Scripts/ClickerGame/Tests/ClickerModelTests.cs`.

## 한글 폰트

- 모든 TMP 텍스트는 생성기가 `Assets/Fonts/Pretendard-Regular SDF.asset`을 로드해 자동 주입한다(`ApplyKoreanFont`). 폰트 에셋이 없으면 텍스트가 □(두부)로 보인다.
- 동봉 폰트: `Pretendard-Regular SDF.asset`·`Pretendard-Bold SDF.asset`(+ 원본 `.ttf`). 새 글자가 깨지면 TMP Font Asset의 글리프 설정(Dynamic) 또는 재생성을 확인한다.

## 문제 해결

- **텍스트가 □로 보임** — `Assets/Fonts/Pretendard-Regular SDF.asset`이 있는지, 생성기 `KoreanFontPath` 상수와 경로가 일치하는지 확인.
- **버튼을 눌러도 반응 없음** — 씬에 `EventSystem`이 있고 `InputSystemUIInputModule`이 붙어 있는지 확인(이 프로젝트는 Input System이라 레거시 `StandaloneInputModule` 금지). 생성기가 자동 추가하지만, 기존 씬에 레거시 모듈이 있으면 충돌한다.
- **패널이 화면 밖** — Canvas는 `ScaleWithScreenSize`(기준 1920×1080). Game 뷰 해상도를 16:9로.
- **씬을 못 찾음** — 씬은 커밋되지 않는다. [자동 생성 가이드](자동%20생성%20가이드.md)로 다시 생성.
