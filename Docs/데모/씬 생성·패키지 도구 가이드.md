# 데모 씬 생성·패키지 도구 가이드

강사용 데모 씬을 **에디터 메뉴로 생성**하는 절차와, 학생 배포용 `.unitypackage`를 **익스포트**하는 절차를 한곳에 모은 강사 전용 문서다. 차시별 시연 대본·조작 순서는 각 차시 **따라하기 강사용** 문서에 있다.

> 연결 문서: [00. 수업 개요](../00.%20수업%20개요.md) · [00. 강사 사전 학습 가이드](../강사용/00.%20강사%20사전%20학습%20가이드.md)

## 핵심 원칙 — 생성기가 정본, 씬은 비커밋

씬 파일(`*.unity`)은 손으로 편집하거나 git에 커밋하지 않는다. 대신 **에디터 스크립트가 씬을 코드로 생성**한다. 생성기가 정본이므로, 씬에 무엇을 바꾸려면 씬이 아니라 **생성기 코드**를 고치고 메뉴로 다시 만든다.

- 강사용 데모 씬은 `Assets/Scenes/`에 만들어지며 **git에 커밋하지 않는다**(`.gitignore`의 `/[Aa]ssets/[Ss]cenes/*.unity` 규칙). 새 환경에서는 메뉴로 매번 재생성한다.
- 학생용 씬은 경로가 달라(`Assets/StudentPackage/Scenes/`) 이 규칙에 걸리지 않으며, **GUID 안정성을 위해 커밋한다**. 패키지 익스포트가 이 씬을 묶는다.
- 저장소는 **강사 전용**이다. 학생은 저장소를 클론하지 않고, 강사가 익스포트한 `.unitypackage`만 임포트한다.

## 실행 전 확인 — R3 설치

메뉴(`Tools > Reactive Study`)가 안 보이거나 컴파일이 안 되면 R3 설치부터 확인한다(설치 절차 전체는 [교안 02 §1](../교안/02.%20R3%20시작하기%20-%20Observable%20만들기.md)).

- **R3 코어(NuGet)** — `Assets/Packages/`에 `R3.<버전>` 폴더와 `R3.dll`이 있는지(NuGetForUnity가 `Assets/packages.config` 기준으로 복원). 메뉴 `NuGet > Restore Packages`로 복원 가능.
- **R3.Unity(UPM)** — `Window > Package Manager`에서 `R3.Unity`가 git URL로 들어와 있는지(`AddTo`·프레임 프로바이더 등 Unity 통합 제공).
- **확인 한 줄** — 아무 스크립트에서 `using R3;`가 컴파일되면 설치 완료.

## 1. 강사용 데모 씬 생성

데모는 **두 씬**이다 — 1부 통합 기초(`ReactiveBasics`, 5섹션 2열)와 2부 클리커(`ReactiveClicker`). 둘을 따로 만들 수도, 한 번에 만들 수도 있다.

### 생성 메뉴

| 메뉴                                                | 생성 씬(`Assets/Scenes/`) | 생성기                                                               |
| --------------------------------------------------- | ------------------------- | -------------------------------------------------------------------- |
| `Tools > Reactive Study > 01. R3 기초 통합 씬 생성` | `ReactiveBasics.unity`    | `Assets/Scripts/ReactiveBasics/Editor/ReactiveBasicsSceneCreator.cs` |
| `Tools > Reactive Study > 02. 클리커(2부) 씬 생성`  | `ReactiveClicker.unity`   | `Assets/Scripts/ClickerGame/Editor/ClickerGameSceneCreator.cs`       |

- 1부 통합 씬의 각 섹션 패널은 챕터별 생성기(`CreateStreamsSceneCreator`·`OperatorsSceneCreator`·`LifetimeSceneCreator`·`BindingAsyncSceneCreator`)의 `Build*Panel`이 정본이며, `ReactiveBasicsSceneCreator`가 이를 2열 위치로 재사용해 한 씬에 조립한다.
- 한 섹션만 단독 씬으로 보고 싶으면 `Tools > Reactive Study > 02~05 …` 개별 메뉴도 남아 있다(강사 편의용, 배포에는 쓰지 않음).

### Create All Scenes (전체 한 번에)

`Tools > Reactive Study > Create All Scenes` — 통합 기초 + 클리커 **두 씬**을 한 번에 생성·갱신한다(`Assets/Editor/AllScenesCreator.cs`). 완료 후 통합 기초 씬(`ReactiveBasics`)을 열어 둔다. 새 환경에서 데모 일체를 준비할 때 이 메뉴 하나면 된다.

### 절차

1. 메뉴를 실행한다(개별 또는 Create All Scenes).
2. 확인 대화상자에서 `생성` — 현재 열린 씬은 닫히며, 저장하지 않은 변경은 먼저 저장 여부를 묻는다.
3. 완료 대화상자에 경로가 뜬다. Play로 동작을 확인한다.

### 재실행 시 동작

- 메뉴를 다시 실행하면 **새 씬을 만들어 같은 경로에 덮어쓴다**. 손으로 고친 내용은 사라지므로, 변경은 생성기 코드에 반영한다.
- 자동화·배치에서는 대화상자 없는 진입점을 직접 호출한다 — `ReactiveBasicsSceneCreator.BuildScene(bool save = true)`·`ClickerGameSceneCreator.BuildScene(...)`, 전체는 `AllScenesCreator.BuildAllScenes()`.

## 2. 씬별 구성 요약

각 씬은 한 Canvas(`ScreenSpaceOverlay`, 기준 1920×1080) + `EventSystem`(+ `InputSystemUIInputModule`) 위에 패널과 데모 컴포넌트를 둔다. UI 참조는 생성기가 `SerializedObject`로 `[SerializeField]` 필드(`m_*`)에 코드로 채우므로 **인스펙터 수동 연결이 불필요**하다. 공유 UI 헬퍼는 `Assets/Scripts/ReactiveBasics/Editor/ReactiveSceneUi.cs`.

### `ReactiveBasics` (1부, 5섹션 2열)

왼쪽 열에 §1~§3, 오른쪽 열에 §4~§5를 배치한다.

| 섹션 / 패널                            | 데모 컴포넌트(`Assets/Scripts/`)                    | 핵심 UI                                                                                         |
| -------------------------------------- | --------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| §1 `Panel1_Create`                     | `ReactiveBasics/CreateStreamsDemo.cs`               | `Range(1, 4)`·`Timer 2초`·`Interval 시작`·`Subject.OnNext`·`event 발생 (FromEvent)` 버튼 + 로그 |
| §2 `Panel2_Operators`                  | `ReactiveBasics/OperatorsDemo.cs`                   | 더블클릭 안내 · 검색 입력창 · `탭` 버튼 · 3연타/쿨다운 안내                                     |
| §3 `Panel3_Lifetime` + `LifetimeProbe` | `ReactiveBasics/LifetimeDemo.cs`(+ `LifetimeProbe`) | `구독 추가`·`모두 해제`·`누수 만들기`·`활성 구독 조회`·`프로브 파괴` 버튼 + 상태/Tracker 텍스트 |
| §4 `Panel4_Binding`                    | `ReactiveBasics/BindingDemo.cs`                     | `피해 -10`·`회복 +10`·`MP -5/+5`·`스킬 사용` + HP/MP/상태                                       |
| §5 `Panel5_Async`                      | `ReactiveBasics/AsyncDemo.cs`                       | `순차 로드 (Sequential)`·`드롭 로드 (Drop)` + 로그(1건당 1.5초)                                 |

### `ReactiveClicker` (2부)

| 패널 / 주요 GameObject                                                                    | 데모 컴포넌트                                          | 핵심 UI                                                                                                                                                   |
| ----------------------------------------------------------------------------------------- | ------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ClickButton`·`FeverButton`·`ComboText`·`FeverStateText`·`StatusPanel`·`ClickerPresenter` | `ClickerGame/ClickerPresenter.cs`(+ `ClickerModel.cs`) | `클릭!`·`피버!` · `StatusPanel`(`GoldText`·`StatsText`·`ClickUpgradeButton` "클릭 강화/비용 10"·`AutoUpgradeButton` "도우미 고용/비용 25"·`EventLogText`) |

- 04 섹션(`Panel3_Lifetime`)에는 의도적으로 **누수 만들기** 버튼이 있다 — `AddTo`/`Dispose` 없는 전역 `Interval` 구독은 오브젝트와 무관하게 영원히 산다. `Window > Observable Tracker`와 `활성 구독 조회`로 눈에 보이게 하는 것이 수명 관리 시연의 핵심이다.
- 클리커는 **완성본**이다(강사용). 학생 배포는 핵심 메서드를 비운 **스켈레톤**으로 나간다(아래 3절). `ClickerModel`은 비-GameObject 순수 C#로 `ClickerPresenter`가 `new`로 생성하고 `OnDestroy`에서 `Dispose`한다. 구매 가능 여부(`CanBuyClickUpgrade`·`CanBuyAutoUpgrade`)는 Model에서 `Gold.CombineLatest(비용)`으로 파생되어 버튼 `interactable`에 그대로 바인딩되므로 Presenter에 분기 로직이 없다.

## 3. 학생 배포 패키지 생성·익스포트

학생은 빈 URP 프로젝트에 R3(NuGetForUnity + R3.Unity git URL)·UniTask(git URL)·Input System을 설치한 뒤, **1부 기초 패키지를 가장 먼저** 임포트하고(공통 폰트 포함) 2부에 들어갈 때 클리커 패키지를 임포트한다. 강사는 두 단계로 이 패키지들을 만든다(`Assets/Editor/StudentPackageCreator.cs`, 메뉴 `Tools > Reactive Study > Student Package`).

### 단계 1 — 학생용 씬 생성

`Tools > Reactive Study > Student Package > Create All Student Scenes` — 강사용 씬을 메모리에서 빌드한 뒤 데모 컴포넌트를 `My*` 타입으로 **교체**해 학생용 씬 2개를 `Assets/StudentPackage/Scenes/`에 생성·저장한다.

| 학생 씬(`Assets/StudentPackage/Scenes/`) | 스왑되는 컴포넌트                                                                                                                                                                                               | 학생 코드 형태           |
| ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------ |
| `ReactiveBasics.unity`                   | `CreateStreamsDemo`→`MyCreateStreamsDemo` · `OperatorsDemo`→`MyOperatorsDemo` · `LifetimeDemo`→`MyLifetimeDemo`(+`LifetimeProbe`→`MyLifetimeProbe`) · `BindingDemo`→`MyBindingDemo` · `AsyncDemo`→`MyAsyncDemo` | **스켈레톤**(라이브코딩) |
| `ReactiveClicker.unity`                  | `ClickerPresenter`→`MyClickerPresenter`(+ `MyClickerModel`)                                                                                                                                                     | **스켈레톤**(라이브코딩) |

- 1·2부 모두 핵심 R3 메서드 본문을 비운 **스켈레톤**으로 배포한다(주석 제거, 라이브 코딩). 1부는 각 데모의 `Start()`/`Setup*` 본문이 비어 있고(필드·표시 헬퍼·생명주기는 제공), 2부(클리커)는 `MyClickerModel` 생성자의 `CombineLatest` 파생·자동수입 `Interval`·`TryStartFever`, `MyClickerPresenter`의 `BindInput`/`BindView`가 비어 있어 수업 중 라이브 코딩으로 채운다.
- 학생 스크립트는 `Assets/StudentPackage/Scripts/`에 있다(`MyLifetimeProbe`는 `MyLifetimeDemo.cs` 안에 함께 정의되어 있어 1부 패키지가 `MyLifetimeDemo.cs` 하나만 묶어도 된다).

> ⚠️ **SwapComponent 필드명 동기화** — 스왑은 강사용 컴포넌트의 직렬화 필드(`m_*`)를 같은 이름으로 `My*` 컴포넌트에 복사한다. `My*` 쪽에 대응 필드가 없으면 _"`MyXxx`에 '`m_...`' 필드가 없습니다 - 원본과 필드명을 동기화하세요"_ 예외로 생성이 중단된다. 강사용 데모의 `[SerializeField]`이름을 바꿨다면`My\*` 스크립트의 필드명도 똑같이 맞춰야 한다.

### 단계 2 — 패키지 익스포트

학생 씬을 만든 뒤 `.unitypackage`로 내보낸다.

- `Tools > Reactive Study > Student Package > Export All Student Packages` — 폴더를 고르면 아래 2종을 한 번에 익스포트한다.
- `Tools > Reactive Study > Student Package > Export 1. Basics` / `Export 2. Clicker` — 개별 익스포트(파일 저장 경로를 직접 지정).

| 패키지 파일                   | 담기는 에셋                                                                                                                              |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `R3Lecture-Student-1-Basics`  | `Assets/Fonts`·`Assets/TextMesh Pro`(Pretendard 한글 폰트 + TMP 설정) · `ReactiveBasics.unity` + `MyCreateStreamsDemo`~`MyAsyncDemo` 5종 |
| `R3Lecture-Student-2-Clicker` | `ReactiveClicker.unity` + `MyClickerModel.cs` + `MyClickerPresenter.cs`                                                                  |

- **1부 기초 패키지를 가장 먼저** — 한글 깨짐을 막으려면 학생이 폰트·TMP 설정부터 임포트해야 한다(폰트는 1부 패키지에만 들어 있다).
- 익스포트는 넣을 에셋이 실제로 존재하는지 먼저 검사한다. 없으면 _"패키지에 넣을 에셋이 없습니다 ... 학생용 씬을 먼저 생성했는지(Create All Student Scenes) ... 확인하세요"_ 예외가 난다 — **단계 1을 건너뛰지 말 것**.
- 자동화·배치는 `StudentPackageCreator.BuildAndExportBatch()`(씬 생성 후 환경변수 `STUDENT_PACKAGE_OUTPUT` 폴더로 전체 익스포트)를 호출한다.

## 4. 한글 폰트 (Pretendard 자동 주입)

- 모든 TMP 텍스트는 생성기가 `Assets/Fonts/Pretendard-Regular SDF.asset`을 로드해 자동 주입한다(`ReactiveSceneUi.ApplyKoreanFont`, 경로 상수 `KoreanFontPath`). 폰트 에셋이 없으면 텍스트가 □(두부)로 보인다.
- 동봉 폰트: `Pretendard-Regular SDF.asset`·`Pretendard-Bold SDF.asset`(+ 원본 `.ttf`). 새 글자가 깨지면 TMP Font Asset의 글리프 설정(Dynamic) 또는 재생성을 확인한다.
- 학생 쪽은 `R3Lecture-Student-1-Basics`(폰트 + TMP 설정)을 가장 먼저 임포트해 같은 폰트를 확보한다.

## 5. 문제 해결

- **`Tools > Reactive Study` 메뉴가 없음** — 에디터 스크립트가 컴파일되지 않은 것. Console의 컴파일 에러부터 해결(R3 미복원이면 위 [실행 전 확인](#실행-전-확인--r3-설치)).
- **텍스트가 □(두부)** — `Assets/Fonts/Pretendard-Regular SDF.asset` 누락 또는 경로 불일치(생성기 `KoreanFontPath` 상수와 비교). 학생 쪽이면 1부 기초 패키지 미임포트.
- **버튼을 눌러도 반응 없음** — 씬에 `EventSystem`이 있고 `InputSystemUIInputModule`이 붙어 있어야 한다(이 프로젝트는 Input System이라 레거시 `StandaloneInputModule` 금지). 생성기가 자동 추가하지만, 기존 씬에 레거시 모듈이 있으면 충돌하므로 제거한다.
- **패널이 화면 밖** — Canvas는 `ScaleWithScreenSize`(기준 1920×1080). Game 뷰 해상도를 16:9로.
- **씬을 못 찾음** — 강사용 씬은 커밋되지 않는다(생성기가 정본). 위 [강사용 데모 씬 생성](#1-강사용-데모-씬-생성)으로 다시 생성한다. 학생 씬은 패키지에 들어 있으므로 해당 패키지를 임포트한다.
- **`My*`에 'm\_...' 필드가 없음(스왑 실패)** — `My*` 스크립트의 직렬화 필드명을 원본 강사용 컴포넌트와 동기화한다(위 [SwapComponent 주의](#단계-1--학생용-씬-생성)).
- **익스포트 시 "넣을 에셋이 없습니다"** — 학생용 씬을 먼저 만들지 않았거나 파일이 이동·개명된 것. `Create All Student Scenes`를 먼저 실행한다.
- **NuGet/R3 관련 에러** — `Assets/packages.config`에 `R3`가 있는지, `Assets/Packages/`에 복원됐는지 확인 후 `NuGet > Restore Packages`.
