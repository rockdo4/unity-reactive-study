# 데모 자동 생성 가이드

데모 씬 두 개를 **에디터 메뉴로 생성**하는 절차와 생성물 체크리스트. 씬 구성은 [구성 가이드](구성%20가이드.md), 수업 진행은 [시연 진행 가이드](시연%20진행%20가이드.md)를 참조.

> 연결 문서: [00. 수업 개요](../00.%20수업%20개요.md) · [강사용 운영 가이드](../강사용/00.%20강의%20운영%20가이드.md) · 자매 문서 [구성 가이드](구성%20가이드.md) · [시연 진행 가이드](시연%20진행%20가이드.md)

## 개요

씬 파일(`*.unity`)은 git에 커밋하지 않는다. 대신 **에디터 스크립트가 씬을 코드로 생성**한다 — 생성기가 정본이므로 코드가 바뀌면 씬을 다시 만든다.

| 메뉴                                             | 생성 씬                               | 생성기                                                               |
| ------------------------------------------------ | ------------------------------------- | -------------------------------------------------------------------- |
| `Tools > Reactive Study > 1부 기초 데모 씬 생성` | `Assets/Scenes/ReactiveBasics.unity`  | `Assets/Scripts/ReactiveBasics/Editor/ReactiveBasicsSceneCreator.cs` |
| `Tools > Reactive Study > 2부 클리커 씬 생성`    | `Assets/Scenes/ReactiveClicker.unity` | `Assets/Scripts/ClickerGame/Editor/ClickerGameSceneCreator.cs`       |

## 실행 전 확인 — R3 설치

이 저장소는 R3가 **이미 설치**되어 있다. 새 환경에서 메뉴가 안 보이거나 컴파일이 안 되면 아래를 확인한다(설치 절차 전체는 [교안 02 §1](../교안/02.%20R3%20시작하기%20—%20Observable%20만들기.md)).

- **R3 코어(NuGet)** — `Assets/Packages/`에 `R3.<버전>` 폴더와 `R3.dll`이 있는지(NuGetForUnity가 `Assets/packages.config` 기준으로 복원). 메뉴 `NuGet > Restore Packages`로 복원 가능.
- **R3.Unity(UPM)** — `Window > Package Manager`에서 `R3.Unity`가 git URL로 들어와 있는지(`AddTo`·프레임 프로바이더 등 유니티 통합 제공).
- **확인 한 줄** — 아무 스크립트에서 `using R3;`가 컴파일되면 설치 완료.

## 1부 기초 데모 씬 생성

1. 메뉴 `Tools > Reactive Study > 1부 기초 데모 씬 생성`.
2. 확인 대화상자에서 `생성`(현재 씬은 닫히며, 저장 안 한 변경은 먼저 저장 여부를 물음).
3. 완료 대화상자에 경로(`Assets/Scenes/ReactiveBasics.unity`)가 뜬다. Play로 패널 1~5를 확인.

### 생성물 체크리스트

- [ ] `Canvas`(ScreenSpaceOverlay, 1920×1080 기준) + `EventSystem`(+ `InputSystemUIInputModule`)
- [ ] `Panel1_Create` + `CreateStreamsDemo`(버튼 5개·로그 연결)
- [ ] `Panel2_Operators` + `OperatorsDemo`(더블클릭·검색·탭·쿨다운)
- [ ] `Panel3_Lifetime` + `LifetimeDemo` + `LifetimeProbe`(구독 추가/해제/누수/Tracker/프로브)
- [ ] `Panel4_Binding` + `BindingDemo`(HP·MP·상태·스킬)
- [ ] `Panel5_Async` + `AsyncDemo`(Sequential·Drop)
- [ ] 모든 텍스트가 한글로 보임(Pretendard 자동 주입) — □면 [문제 해결](#문제-해결)

## 2부 클리커 씬 생성

1. 메뉴 `Tools > Reactive Study > 2부 클리커 씬 생성`.
2. `생성` → 완료 대화상자에 `Assets/Scenes/ReactiveClicker.unity`.
3. Play로 클릭·콤보·피버·업그레이드·자동수입을 확인.

### 생성물 체크리스트

- [ ] `Canvas` + `EventSystem`(+ `InputSystemUIInputModule`)
- [ ] `ClickButton`("클릭!")·`FeverButton`("피버!")·`ComboText`·`FeverStateText`
- [ ] `StatusPanel`(`GoldText`·`StatsText`·`ClickUpgradeButton`·`AutoUpgradeButton`·`EventLogText`)
- [ ] `ClickerPresenter` — 위 UI 참조가 코드로 자동 연결됨(인스펙터 수동 연결 불필요)
- [ ] Play 시 클릭→골드 증가, 도우미 고용 후 초당 자동수입 증가

> 필드 자동 연결: 생성기가 `SerializedObject`로 `m_ClickButton` 등 `[SerializeField]` 참조를 코드에서 채운다. 그래서 생성 직후 인스펙터에서 따로 드래그하지 않아도 동작한다.

## 재실행 시 동작

- 메뉴를 다시 실행하면 **새 씬을 만들어 같은 경로에 덮어쓴다**(`NewScene` → `SaveScene`). 손으로 고친 내용은 사라지므로, 변경은 생성기 코드에 반영한다.
- 자동화/배치에서는 대화상자 없는 `CreateScene()`을 직접 호출할 수 있다(각 생성기의 `public static void CreateScene()`).

## 문제 해결

- **`Tools > Reactive Study` 메뉴가 없음** — 에디터 스크립트가 컴파일되지 않은 것. Console의 컴파일 에러부터 해결(R3 미복원이면 위 [실행 전 확인](#실행-전-확인--r3-설치)).
- **텍스트가 □(두부)** — `Assets/Fonts/Pretendard-Regular SDF.asset` 누락 또는 경로 불일치(생성기 `KoreanFontPath` 상수).
- **버튼 무반응** — `EventSystem`에 `InputSystemUIInputModule`이 붙어야 함(Input System 프로젝트). 기존 씬의 레거시 `StandaloneInputModule`은 제거.
- **NuGet/R3 관련 에러** — `Assets/packages.config`에 `R3`가 있는지, `Assets/Packages/`에 복원됐는지 확인 후 `NuGet > Restore Packages`.
