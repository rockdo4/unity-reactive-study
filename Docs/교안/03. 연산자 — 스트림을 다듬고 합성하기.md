# 03. 연산자 — 스트림을 다듬고 합성하기

리액티브의 **진짜 가치**가 나오는 차시. 스트림을 거르고(`Where`), 바꾸고(`Select`), 시간으로 다듬고(`Debounce`/`ThrottleFirst`/`Chunk`), 여러 스트림을 합쳐(`Merge`/`CombineLatest`) **의도를 그대로 코드로 적는** 법을 익힘.

> 따라하기 데모: 기초 데모 씬 **패널 2 "연산자"** · 코드 `Assets/Scripts/ReactiveBasics/OperatorsDemo.cs`

## 1. 연산자란 — 스트림 파이프라인

연산자(operator)는 **Observable을 받아 새 Observable을 돌려주는 메서드**. 그래서 메서드 체인으로 파이프라인을 만들 수 있음.

```csharp
source                      // 원본 스트림
    .Where(x => x > 0)      // 거르고
    .Select(x => x * 10)    // 바꾸고
    .Subscribe(Print);      // 마지막에 구독
```

```
source:  ──(-1)──2──0──3──▶
Where:   ───────2─────3──▶     (0보다 큰 것만)
Select:  ───────20────30─▶     (×10)
```

원칙 하나 — **구독은 체인의 맨 끝에서 한 번.** 중간 단계는 전부 "설계도"고, `Subscribe`가 호출돼야 물이 흐름.

## 2. 연산자 카탈로그 — 전부 외우지 말고 분류만

연산자는 수십 개지만 **용도별 그룹**만 잡으면 필요할 때 찾아 쓰면 됨.

| 그룹          | 대표 연산자 (R3 기준)                                             | 하는 일                      |
| ------------- | ----------------------------------------------------------------- | ---------------------------- |
| **생성**      | `Range` · `Timer` · `Interval` · `FromEvent` · `EveryUpdate`      | 스트림을 만듦 (2차시)        |
| **필터**      | `Where` · `Distinct` · `DistinctUntilChanged` · `Take` · `Skip`   | 조건·개수로 걸러냄           |
| **변환**      | `Select` · `Cast` · `Index`                                       | 값을 다른 모양으로           |
| **시간**      | `Debounce` · `ThrottleFirst` · `ThrottleLast` · `Delay` · `Chunk` | 시간으로 다듬음              |
| **결합**      | `Merge` · `CombineLatest` · `Zip` · `WithLatestFrom` · `Concat`   | 여러 스트림을 합침           |
| **누적**      | `Scan` · `Aggregate`                                              | 흘러온 값을 누적함           |
| **에러/완료** | `OnErrorResume` 처리 · `Catch` · `OnErrorResumeAsFailure`         | 에러·완료를 다룸 (5차시)     |
| **공유**      | `Share` · `Replay` 계열                                           | 하나의 구독을 여럿이 나눠 씀 |

## 3. 필터와 변환 — Where · Select · DistinctUntilChanged · Scan

LINQ와 같은 이름, 같은 의미. 대상이 "시간 위의 스트림"일 뿐.

```csharp
// 10점 단위 점수만 통과시켜 마일스톤 알림
playerStat.OnScoreChanged
    .Where(score => score % 10 == 0)
    .Subscribe(score => Debug.Log($"마일스톤! {score}"));

// HP를 "구간 라벨"로 변환하고, 구간이 바뀔 때만 갱신
vitals.Hp
    .Select(hp => hp < 30 ? "위험" : hp < 70 ? "주의" : "안전")
    .DistinctUntilChanged()                 // 같은 라벨이 연속되면 무시
    .Subscribe(state => m_StateText.text = state);
```

`DistinctUntilChanged`는 **연속 중복 제거** — UI 갱신·사운드 재생처럼 "바뀔 때만 한 번" 반응해야 하는 곳의 단골임.

`Scan`은 "지금까지의 누적"을 흘려보냄(LINQ `Aggregate`의 실시간 버전):

```csharp
// 획득 골드 스트림 → 누적 합계 스트림
goldGained.Scan(0L, (total, gained) => total + gained)
    .Subscribe(total => m_TotalText.text = $"누적 {total}");
```

```
입력:  ──5───3───10──▶
Scan:  ──5───8───18──▶    (seed 0에서 시작해 누적)
```

## 4. 시간 연산자 — Debounce · ThrottleFirst · Chunk

시간 연산자가 리액티브의 백미. **상태 플래그와 타이머를 직접 들지 않고** 시간 조건을 선언함.

### Debounce — 잠잠해질 때까지 기다리기

마지막 이벤트 후 지정 시간 동안 새 이벤트가 없으면, 그 마지막 값을 내보냄.

```
입력:     ──a─b──c──────────d─────▶
Debounce(0.5s):
출력:     ────────────c──────────d▶   (연타 중에는 침묵, 잠잠해지면 마지막 값)
```

```csharp
// 검색창: 타이핑이 0.5초 멈췄을 때만, 직전과 다른 검색어일 때만 검색 실행
m_SearchInput.onValueChanged.AsObservable()
    .Debounce(TimeSpan.FromMilliseconds(500))
    .DistinctUntilChanged()
    .Subscribe(query => Search(query));
```

쓰임: 검색 자동완성, 설정 자동 저장, 리사이즈 후 재배치 — "**입력이 끝났을 때** 한 번".

### ThrottleFirst — 첫 발만 통과, 구간 무시

첫 이벤트를 통과시키고, 지정 시간 동안 나머지를 무시함.

```
입력:     ──a─b─c────d─e───────f──▶
ThrottleFirst(1s):
출력:     ──a────────d──────────f─▶   (발동 직후 1초는 귀를 닫음)
```

```csharp
// 스킬 버튼: 첫 클릭만 발동, 2초 쿨다운
m_SkillButton.OnClickAsObservable()
    .ThrottleFirst(TimeSpan.FromSeconds(2))
    .Subscribe(_ => CastSkill());
```

쓰임: 스킬 쿨다운, 연타 방지, 공격 입력 제한 — "**먼저 발동하고** 잠시 무시".

> 💡 헷갈리면 이렇게 — **Debounce는 "끝나면", ThrottleFirst는 "먼저"**. 같은 연타 입력에 Debounce는 마지막에 한 번, ThrottleFirst는 처음에 한 번 반응함.

### Chunk — 모아서 묶기

이벤트를 모았다가 **경계**(시간·개수·다른 스트림)가 오면 배열로 내보냄.

```csharp
// 500ms 안에 3번 클릭 → 3연타
taps.Chunk(TimeSpan.FromMilliseconds(500), 3)
    .Where(xs => xs.Length >= 3)
    .Subscribe(_ => Debug.Log("3연타!"));

// 더블클릭: "클릭이 250ms 잠잠해질 때"를 경계로 클릭을 묶고, 2개 이상이면
var clicks = Observable.EveryUpdate()
    .Where(_ => Mouse.current != null && Mouse.current.leftButton.wasPressedThisFrame)
    .Share();                                            // ← 7장에서 설명

clicks
    .Chunk(clicks.Debounce(TimeSpan.FromMilliseconds(250)))   // 경계 = 잠잠해진 순간
    .Where(xs => xs.Length >= 2)
    .Subscribe(_ => Debug.Log("더블 클릭"));
```

1차시의 명령형 더블클릭(코루틴+플래그+타이머)과 비교 — "클릭을 모아서, 2개 이상이면"이라는 **의도가 그대로 코드**가 됨.

## 5. 결합 연산자 — Merge · CombineLatest

### Merge — 여러 스트림을 한 줄로

```
A:      ──a─────c─────▶
B:      ─────b─────d──▶
Merge:  ──a──b──c──d──▶    (도착 순서대로 한 스트림에)
```

```csharp
// 키보드 스페이스든 버튼 클릭이든 "발사"는 발사
var fireByKey = Observable.EveryUpdate()
    .Where(_ => Keyboard.current != null && Keyboard.current.spaceKey.wasPressedThisFrame)
    .Select(_ => Unit.Default);
var fireByButton = m_FireButton.OnClickAsObservable();

fireByKey.Merge(fireByButton)
    .ThrottleFirst(TimeSpan.FromSeconds(0.5))   // 합친 뒤 공통 쿨다운까지
    .Subscribe(_ => Fire());
```

### CombineLatest — 각 스트림의 "최신값" 조합

어느 쪽이든 새 값이 오면, **모든 스트림의 최신값**을 모아 내보냄.

```
HP:              ──100────70──────────30────▶
MP:              ──50──────────10───────────▶
CombineLatest:   ──(100,50)─(70,50)─(70,10)─(30,10)▶
```

```csharp
// HP가 있고 MP가 충분할 때만 스킬 버튼 활성화 — "조건이 여러 값에 걸칠 때"의 정석
hp.CombineLatest(mp, (h, m) => h > 0 && m >= 20)
    .DistinctUntilChanged()
    .Subscribe(canUse => m_SkillButton.interactable = canUse);
```

`Zip`(짝이 맞을 때만)·`WithLatestFrom`(한쪽이 주도)은 카탈로그에서 찾아 쓰면 됨 — 게임 UI의 9할은 `CombineLatest`로 충분함.

## 7. Share — 핫/콜드 혼동 방지

더블클릭 코드에서 `clickStream`을 **본체와 Debounce 경계 두 곳**이 씀. `Share()`가 없으면 두 곳이 **각자 따로 구독**해서 `EveryUpdate` 검사가 두 벌 돌게 됨(콜드하게 동작). 값비싼 스트림(네트워크·무거운 계산)이라면 이게 버그가 됨.

```csharp
var clicks = rawClicks.Share();   // 원본을 한 번만 구독하고, 구독자들이 그 결과를 나눠 받음
```

규칙: **한 스트림을 두 곳 이상에서 참조하면 일단 `Share()`를 의심**할 것.

## 8. 부록 — 옛 UniRx 코드를 읽을 때 (이름 매핑)

인터넷 튜토리얼의 다수가 UniRx 기준임. R3는 연산자 이름을 .NET LINQ 표준에 맞춰 바꿨으므로 **그대로 붙이면 컴파일이 안 됨.**

| UniRx                           | R3                            | 함정/메모                                                             |
| ------------------------------- | ----------------------------- | --------------------------------------------------------------------- |
| `Buffer`                        | `Chunk`                       | 결과가 `IList<T>`(`.Count`) → **`T[]`(`.Length`)** — 개수 검사도 바뀜 |
| `Throttle`                      | `Debounce`                    | **최대 함정** — R3엔 `Throttle`이 아예 없음                           |
| `Sample`                        | `ThrottleLast`                |                                                                       |
| `StartWith`                     | `Prepend`                     |                                                                       |
| `ThrottleFirst`                 | `ThrottleFirst`               | 이름·동작 그대로                                                      |
| `IObservable<T>`/`IObserver<T>` | `Observable<T>`/`Observer<T>` | 인터페이스 → 추상 클래스                                              |
| `this.UpdateAsObservable()`     | `Observable.EveryUpdate()`    | R3 전역 스트림은 수명 관리 필수(4차시)                                |

핵심 프레임: **개념은 그대로, 이름이 LINQ 표준에 맞춰 바뀌었다.**

## 9. 정리 & 체크리스트

- 연산자 = Observable → Observable 함수. 체인으로 파이프라인을 만들고 **구독은 맨 끝 한 번**.
- 분류만 외움: 생성·필터·변환·시간·결합·누적·에러·공유.
- 시간 3총사 — **Debounce("끝나면")** · **ThrottleFirst("먼저")** · **Chunk("모아서")**.
- 결합 2총사 — **Merge(한 줄로)** · **CombineLatest(최신값 조합 → 버튼 활성화의 정석)**.
- 한 스트림을 여러 곳에서 쓰면 **`Share()`**.
- UniRx → R3 이름 매핑(`Buffer→Chunk`, `Throttle→Debounce`)을 모르면 옛 자료에 당함.

- [ ] 더블클릭 파이프라인(`Chunk` + `Debounce` 경계)을 한 문장으로 읽을 수 있음

- [ ] `Share()`가 필요한 상황(한 스트림을 두 곳에서 참조)을 식별할 수 있음
- [ ] UniRx의 `Throttle`이 R3의 무엇인지 즉답할 수 있음(`Debounce`)
